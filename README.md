# Knowledge World

> 不是学习知识，而是解锁认知地图。

Knowledge World 是一个基于 What / How / Why / System 四层认知模型和像素风探索式交互的 AI 学习系统。它将学习资料转化为四张逐层深入的知识地图，用户先收集事实与概念，再通过提问、反馈和追问把知识推进到机制、原因与系统迁移。

参赛赛事：**TRAE AI 创造力大赛 2026**（赛题：世界很大，放手去造）

## 项目定位

让学习像探索未知世界一样发生。系统把 PDF / 文章 / 课程转化为 What / How / Why / System 四张深度地图；同一批知识节点在四张地图里坐标不变，用户以学者化身反复抵达同一个节点，看到自己对它的理解越来越深。

## 核心理念

- 知识不是内容，而是世界
- 学习不是阅读，而是探索
- AI 不是讲解者，而是认知引导者

## 认知模型

| 层级 | 含义 | 产品表达 |
|---|---|---|
| What | 这是什么 | 翻卡阅读，收集定义、例子和连接 |
| How | 它如何运作 | 机制问答、步骤补全、因果链理解 |
| Why | 为什么如此 | 反事实追问、原因解释、底层条件判断 |
| System | 如何与其他知识连接 | 跨节点迁移、现实应用、反例讨论 |

四层不是一张地图里的四个区域，而是四张独立深度地图。用户可以随时回到低层地图；进入更高层需要先完成前一层的至少一个节点。

## AI Cognitive Engine

AI 不是聊天，而是在做三件事：

1. 判断用户处于哪个认知层级
2. 找出当前最大理解误区
3. 生成下一步最优问题（Socratic Path）

这让 Knowledge World 和普通 ChatGPT 对话区分开来：系统不是回答用户问题，而是持续制造能推动理解前进的问题。

## MVP 三件事

1. 进入《人类简史》预制世界 → 看到 What 地图、起始谜题和结构性迷雾
2. 点击节点 → 学者小人移动到 NPC 旁边 → What 层翻三张知识卡
3. 切换到 How / Why / System → 回答 NPC 的高阶问题 → AI 诊断反馈 → 迷雾消散并露出邻居节点

## 学习状态系统

| 状态 | 地图表现 | 含义 |
|---|---|---|
| locked | 被迷雾完全隐藏 | 当前深度尚未解锁 |
| available | NPC 和谜题可见 | 用户可以点击进入 |
| learning | 弱光提示 | 高阶问答中，用户还没完成当前深度 |
| completed | 节点亮起 | 当前节点在当前深度已完成 |

每个节点分别记录 What / How / Why / System 四个深度的状态。HUD 使用"认知迷雾已消散 X%"表达进度，避免把学习体验变成考试分数。

## 核心创新

- **四张深度地图**：同一批节点在 What / How / Why / System 中反复出现，用户感知到理解逐层加深
- **What 不主做问答**：事实和概念先通过翻卡被收集，降低一开始就被考试的压力
- **How / Why / System 问答推进**：AI 不一上来灌输答案，而是根据用户回答给出反馈、提示和追问
- **知识迷雾系统**：未解锁节点在当前深度完全隐藏，完成节点后迷雾消散并露出邻居谜题
- **NPC 谜题驱动**：地图上显示问题而不是知识点名称，让用户因为好奇而点击节点
