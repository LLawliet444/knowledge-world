# Knowledge World

> 不是学习知识，而是解锁认知地图。

Knowledge World 是一个基于《如何阅读一本书》认知模型 + 像素风探索式交互的 AI 学习系统。它将任何学习资料转化为可探索的知识世界，用户通过任务、提问与反馈逐步提升认知层级，而不是被动阅读内容。

参赛赛事：**TRAE AI 创造力大赛 2026**（赛题：世界很大，放手去造）

## 项目定位

让学习像探索未知世界一样发生。系统把 PDF / 文章 / 课程转化为一张像素风世界地图，用户以学者化身在地图上移动，经过 AI 提问、诊断反馈和迷雾解锁，逐步完成从 What 到 System 的理解跃迁。

## 核心理念

- 知识不是内容，而是世界
- 学习不是阅读，而是探索
- AI 不是讲解者，而是认知引导者

## 认知模型

| 层级 | 含义 | 产品表达 |
|---|---|---|
| What | 这是什么 | 概念入口、基础节点 |
| How | 结构是什么 | 知识模块、路径关系 |
| Why | 为什么如此 | 因果、原理、挑战节点 |
| System | 如何与其他知识连接 | 跨领域连接、开放区域 |

## AI Cognitive Engine

AI 不是聊天，而是在做三件事：

1. 判断用户处于哪个认知层级
2. 找出当前最大理解误区
3. 生成下一步最优问题（Socratic Path）

这让 Knowledge World 和普通 ChatGPT 对话区分开来：系统不是回答用户问题，而是持续制造能推动理解前进的问题。

## MVP 三件事

1. 上传 PDF → 自动生成像素风四层世界地图（≤20 个节点）
2. 点击节点 → 学者小人移动 + 触发 AI 苏格拉底提问对话框
3. 用户回答 → AI 诊断理解缺口 → 输出反馈卡 / 必要讲解 → 迷雾消散动画 + 邻接节点解锁

## 学习状态系统

| 状态 | 地图表现 | 含义 |
|---|---|---|
| 未探索 | 灰色迷雾 | 用户尚未进入该节点 |
| 已访问 | 节点可见 | 用户到达过，但未完成理解 |
| 理解中 | 黄色弱光 | 用户有部分理解，还需要提示或追问 |
| 已掌握 | 绿色亮光 | 用户已理解核心概念与逻辑 |
| 迁移应用 | 紫色光环 | 用户能将知识迁移到新情境 |

## 核心创新

- **从"学习内容"到"认知世界"**：知识是被探索的，不是被消费的
- **先提问，再反馈**：AI 不一上来灌输答案，而是先诊断理解，再给提示、反馈或最小讲解
- **知识迷雾系统**：未掌握内容以视觉迷雾呈现，解锁带来强反馈
- **四层认知模型**：What → How → Why → System，覆盖《如何阅读一本书》的全部维度
- **空间化学习体验**：把线性学习路径变成可导航、可回看、可扩展的认知地图
