# Knowledge World 产品需求文档（MVP）

> 不是学习知识，而是解锁认知地图。

文档创建日期：2026-06-16
项目代号：Knowledge World
赛事归属：TRAE AI 创造力大赛 2026

---

## 0. 初赛版本范围（参赛 Demo 优先级）

为了让初赛 Demo 在 30 秒内击中评委，本节明确锁死本版本要做、可选做、不做的范围。开发期间一切优先级冲突以本节为准。

### 0.1 初赛必须完成（P0，决定产品成立）

- 至少 1 个预制世界（《人类简史》）完整跑通 What / How / Why / System 四张深度地图
- 同一批知识节点在四张地图中保持坐标不变，用户能看到同一个知识点被逐层理解
- 节点不显示节点名，显示关卡 NPC 头像 + 一句谜题标题，驱动用户点击下一个节点
- 关卡 NPC 只承担两件事：初次展示原始谜题；四层完成后再次提出同一个原始谜题作为最终终问
- 关卡首次引入使用 5-8 秒小场景：展示一个与知识点强相关的视觉矛盾，由关卡 NPC 提出 `mysteryQuestion`，随后老学者接管进入 What 翻卡
- 小场景只在用户首次进入某节点 What 层时自动播放；之后再次访问同一节点不自动播放，防止回访时打断用户
- P0 阶段至少覆盖《人类简史》3 个核心节点的小场景：认知革命、农业革命、货币
- 老学者是全程导师 NPC，负责 What 翻卡、How / Why / System 提问、诊断反馈、层级切换引导和终问前的回到原点提示
- What 层采用老学者翻卡阅读：定义、例子、连接三张卡，不做答题
- How / Why / System 层由老学者提问：用户输入回答，AI 输出"已理解 / 还缺一点 / 提示 / 下一问"四段式反馈
- 节点点击 → 学者化身像素小人移动到关卡 NPC 旁 → 老学者对话框弹出阅读卡或 AI 提问
- 节点四层完成后，触发"原问回响"：系统引导用户回到 What 地图，由关卡 NPC 再次提出同一个 `mysteryQuestion`
- 迷雾结构性隐藏节点：未解锁节点不可见，完成节点后迷雾消散并露出邻居谜题
- 地图角落提供深度切换器，可视化"已完成 / 可进入 / 锁定"三种状态

### 0.2 初赛尽量完成（P1，加分项，做不完不影响演示）

- PDF 上传 → 自动生成世界（路演时作为"能力展示"短镜头，不作为主链路入口）
- 第二个预制世界（《经济学原理》第一章）
- 区域征服徽章
- 小场景覆盖全部预制节点

### 0.3 初赛不展示或弱化（P2，留到复赛）

- 世界列表 / 多世界切换
- 完整埋点体系（初赛只保留 `intro_scene_viewed`、`what_card_completed`、`depth_switched`、`depth_completed`、`final_question_completed` 与 `session_complete`）
- 长期学习档案、积分、成就系统
- 申诉按钮、多模型协作

> 初赛阶段所有"上传任意 PDF 自动生成"的演示都从预制世界进入，避免现场生成等待与质量不可控带来的风险。

## 1. 需求背景

### 1.1 触发来源

AI 学习工具在 2024-2026 年迎来爆发，但绝大多数产品只解决了"信息获取"问题——总结、问答、生成课程——本质仍是把 AI 当作更聪明的"讲解员"。学习者拿到的依然是被动消费的内容，而非主动构建的理解。

### 1.2 用户当前现状与痛点

| 痛点 | 表现 | 后果 |
|---|---|---|
| 内容过载 | 收藏夹里堆满 PDF 与课程，真正读完的不到 10% | 知识焦虑，越学越虚 |
| 被动消费 | 读完一本书或听完一门课，能复述但不会用 | 理解停留在 What 层 |
| 缺乏反馈 | 不知道自己哪里没懂，也没人追问 | 自我评估失真 |
| 学习无趣 | 严肃学习场景缺乏正反馈机制 | 难以坚持 |

证据强度：`[PM 假设 + 行业观察]`，正式版 PRD 上线前需通过 30 人定性访谈验证。

### 1.3 不解决的后果

如果 AI 学习产品继续停留在"总结/讲解"形态，用户会快速对 AI 学习工具产生疲劳，市场窗口被 ChatGPT/Claude 等通用对话产品吃掉。差异化窗口正在快速关闭。

### 1.4 为什么现在可行

大模型已经具备稳定的语义理解、结构抽取、问题生成和反馈生成能力，学习资料可以从"静态内容"转化为"动态认知交互系统"。过去的知识图谱只能组织信息，现在的 AI 可以判断用户的理解层级、定位误区，并持续生成下一步问题，这让探索式学习从界面创意变成可落地的产品机制。

## 2. 产品目标

### 2.1 核心目标

> 让学习像探索未知世界一样发生。

用户把一份学习资料转化为可导航的知识世界。系统先用 What 翻卡帮助用户收集事实与概念，再用 How / Why / System 问答推动用户解释机制、追问原因，并把知识迁移到更大的系统里。

### 2.1.1 产品定位

Knowledge World 的定位不是 AI 学习工具，而是认知探索系统：

```text
Knowledge → World → Exploration → Cognition Training
```

它关注的不是"把知识讲得更快"，而是帮助用户看见自己的理解边界，并通过一次次探索把边界向外推进。

### 2.1.2 核心玩法模型

Knowledge World 的地图不再按四个横向区域划分知识，而是用四张独立地图表达认知深度：

```text
What 地图   ：知道它是什么，收集知识卡
How 地图    ：理解它如何运作，补全机制链条
Why 地图    ：追问它为什么成立，解释因果与反事实
System 地图 ：把它迁移到更大的系统，连接其他知识和现实问题
```

同一批节点会出现在四张地图上，坐标保持不变。用户每切换一次深度，看到的不是另一批知识，而是同一个知识世界变得更深。

### 2.1.3 三条核心规则

四张深度地图的所有玩法都受三条规则约束。这三条规则共同保证"层层深入"的认知节奏：用户既能反复回到浅层夯实理解，又不会跳过必要的认知材料；地图既能保持探索的开放感，又不会一打开就把所有内容暴露给用户。

| 规则 | 定义 | 用户感受 |
|---|---|---|
| 向下自由 | 任意时刻都可以从当前深度切换到任意更浅的深度（如从 Why 回到 What），无惩罚、无门槛，已完成节点保持完成状态 | 想复习随时回去 |
| 向上门禁 | 进入 How 需要 ≥ 1 节点完成 What；进入 Why 需要 ≥ 1 节点完成 How；进入 System 需要 ≥ 1 节点完成 Why | 必须先建立基础再深入 |
| 依赖解锁 | 在深度 D 下，节点 N 可用 当且仅当 存在一个邻居 M 且 M 在 D 层已完成；每张深度地图首次进入时，`startNode` 在该深度自动可用 | 迷雾沿已理解的知识向外扩散 |

这三条规则在 UI 上不直接以条文呈现给用户，而是通过深度切换器的状态表达和迷雾的视觉扩散让用户自然理解。

### 2.1.4 节点数据模型新增字段

为支撑关卡 NPC、老学者和原问回响，世界生成 JSON 在每个节点上必须新增以下字段，作为前端渲染和后端生成的契约：

| 字段 | 类型 | 用途 |
|---|---|---|
| `mysteryQuestion` | string | 地图上代替节点名显示的一句谜题文案，激发点击欲望 |
| `gateNpc` | object | 关卡 NPC：`{ id, title, avatar }`，只负责展示谜题标题和最终终问，不负责中间教学 |
| `introScene` | object | 首次进入关卡的小场景：`{ sceneText, visualHint, durationSec, state }`，只在首次进入该节点 What 层时自动播放 |
| `whatCards` | array(3) | 老学者展示的三张翻卡：`[{ kind: "definition", body }, { kind: "example", body }, { kind: "bridge", body }]`，分别对应"定义 / 例子 / 连接" |
| `mentorPrompts` | object | 老学者文案：`{ whatIntro, how, why, system, finalReturn }`，用于四层引导、提问和终问前回到原点提示 |
| `finalQuestion` | object | 原问回响状态：`{ source: "mysteryQuestion", state }`，终问必须复用 `mysteryQuestion`，不得另生成新问题 |

### 2.1.5 双 NPC 职责边界

角色必须保持清晰分工，避免每个关卡 NPC 都变成小老师，削弱老学者的主线价值。

命名约定：文档中统一使用"老学者"指代贯穿全程的导师角色，使用"关卡 NPC"指代地图上与单个知识节点绑定的守关角色；数据字段统一使用 `gateNpc` 表达关卡 NPC，不再使用 `npc` 或 `mentorNpc`。

| 角色 | 负责 | 不负责 |
|---|---|---|
| 关卡 NPC | 地图上的节点形象、谜题标题、最终终问、通关认可 | What 翻卡、How / Why / System 提问、诊断反馈、层级切换说明 |
| 老学者 | What 翻卡、How / Why / System 提问、AI 反馈、层级切换引导、卡住时提示、终问前回到原点提示 | 代替关卡 NPC 提出原始谜题或最终终问 |

因此，一个节点的体验不是"关卡 NPC 教用户"，而是"关卡 NPC 提出问题，老学者带用户理解这个问题，最后用户回到关卡 NPC 面前回答同一个问题"。

### 2.2 量化目标（参赛 MVP 阶段）

| 维度 | 目标 |
|---|---|
| Demo 演示效果 | 上传 PDF 后 ≤90 秒生成可探索世界地图 |
| 核心交互完成率 | 测试用户首次会话内能完成 ≥1 个节点的 What 翻卡，并进入 How 问答 |
| 视觉记忆点 | 路演评委观看 30 秒后能复述"四张深度地图"和"知识迷雾"机制 |
| 反馈可信度 | How / Why / System 中至少 1 层能跑通 AI 追问 - 用户回答 - 诊断反馈 - 解锁判断 |

### 2.3 非目标（明确不做）

- 不做登录系统（MVP 阶段匿名使用，用户数据存 localStorage）
- 不做长期用户成长体系（积分、段位、皮肤等）
- 不做社交分享（评委 demo 阶段无关）
- 不做多人协作探索
- 不做移动端适配（仅 Web 桌面端）

## 3. 用户与场景

### 3.1 目标用户画像

| 画像 | 描述 | 决策权重 |
|---|---|---|
| **求知型自学者** | 有学习目标但缺方法的成年人，例如刚入门读《人类简史》的读者 | 主要 |
| **应试备考者** | 需要构建知识体系的考研/考证用户 | 次要 |
| **教育评委 / 比赛评审** | 关心创新性与产品完整度 | MVP 阶段优先服务 |

不服务：寻找快速答案的用户、需要权威讲解的用户、儿童/低龄学习者。

### 3.2 核心场景排序

| 优先级 | 场景 | 描述 |
|---|---|---|
| P0 | 进入预制世界 | 用户从首页进入《人类简史》预制世界，落点是 What 地图，看到起始节点的关卡 NPC 和谜题标题，周围被结构性迷雾覆盖 |
| P0 | 关卡小场景引入 | 用户首次进入某节点 What 层时，播放 5-8 秒像素小场景，用视觉矛盾引出关卡 NPC 的原始谜题；之后再次访问同一节点不再自动播放 |
| P0 | What 翻卡探索 | 小场景结束后，老学者接管，带用户阅读"定义 / 例子 / 连接"三张卡 |
| P0 | 深度切换 | 完成 ≥ 1 个 What 节点后，深度切换器中的 How 入口从锁定变为可进入；用户切到 How 地图，背景与色调随之变化，但节点坐标不变 |
| P0 | 高阶问答与反馈 | 用户在 How / Why / System 层回答老学者的问题，AI 诊断理解缺口，给出四段式反馈，并决定当前深度是否完成 |
| P0 | 原问回响 | 单个节点四层完成后，老学者引导用户回到 What 地图；关卡 NPC 再次提出同一个原始谜题，用户用自己的话作答，通过后节点完整通关 |
| P0 | 向下复习 | 用户在 How / Why / System 层任意时刻可向下切回 What 或更浅深度复习已完成节点，进度不丢失 |
| P1 | 区域征服反馈 | 一个节点四个深度全部完成后出现完整光晕；完成一组关键节点后弹出区域徽章 |
| P2 | 切换世界 | 同一用户已生成多个世界，可在世界列表间切换 |

### 3.3 关键用户旅程

#### 3.3.1 首次进入《人类简史》世界

预制世界路径，演示主链路。用户当前没有任何进度，落点是 What 地图。

| 步骤 | 用户动作 | 系统反馈 |
|---|---|---|
| 1 | 从首页点击《人类简史》预制世界 | 进入 What 地图：背景为 What 层暖黄色调；只显示 `startNode`（关卡 NPC + 谜题"是什么让智人和其他动物不同？"）；其余位置全是迷雾；HUD 显示"认知迷雾已消散 0%"；深度切换器显示 What=可进入、How/Why/System=锁定 |
| 2 | 点击起始节点 | 学者化身按路径走到关卡 NPC 旁；如果该节点 `introScene.state = "unseen"`，播放 5-8 秒小场景 |
| 3 | 小场景结束 | 关卡 NPC 说出原始谜题；`introScene.state` 标记为 `seen`；随后老学者出现，对话框进入 reading 模式 |
| 4 | 翻完老学者给出的三张卡（定义 / 例子 / 连接），点击"收入图鉴，继续探索" | 该节点 What 深度完成；迷雾从该节点向邻居扩散，露出邻居节点的谜题与关卡 NPC；HUD 数值轻微跳动；深度切换器中的 How 入口由锁定变为可进入 |
| 5 | 继续点击新露出的节点，完成 1-2 个 What | 新节点首次进入时播放各自小场景；已看过的小场景不再自动播放 |
| 6 | 点击深度切换器中的 How 入口 | 触发深度切换动画：背景从暖黄过渡到金绿，节点坐标不变，已完成 What 的节点在 How 地图上变为"可进入"；其它节点仍处于迷雾下 |
| 7 | 点击一个 How 可用节点 | 老学者对话框进入 question 模式，给出 1 个核心提问 + 2 个备用追问；用户输入回答 |
| 8 | 提交回答 | AI 输出四段反馈卡（已理解 / 还缺一点 / 提示 / 下一问）；当回答覆盖 How 核心要点时，节点 How 深度完成，迷雾再次扩散 |
| 9 | 完成同一节点的 What / How / Why / System | 老学者提示"现在回到最初的问题"；系统切回 What 地图该节点位置；不重播小场景；关卡 NPC 再次提出同一个 `mysteryQuestion`；用户回答后，该节点完整通关 |

#### 3.3.2 回访场景：向下复习与跨深度切换

老用户进入已有进度的世界，演示"向下自由 + 进度保持"。

| 步骤 | 用户动作 | 系统反馈 |
|---|---|---|
| 1 | 选择已有世界进入 | 自动落到上次离开时的深度地图，同一节点坐标，已完成节点保留完成态 |
| 2 | 点击深度切换器的 What 入口 | 立即切到 What 地图，已完成节点显示完成光晕，新增节点可继续翻卡，无任何门禁拦截 |
| 3 | 切回 How 地图 | 进度完整保留；之前已完成的 How 节点仍显示完成态；当前深度的迷雾结构与离开时一致 |
| 4 | 完成 ≥ 1 个 Why 节点 | 深度切换器中的 System 入口从锁定变为可进入 |

### 3.4 使用前提

- 用户提供合法的学习资料文件（PDF / TXT / Markdown，单文件 ≤20MB）
- 用户具备基本中文阅读与输入能力
- 浏览器支持 Canvas 2D；WebGL / PixiJS 可作为复赛阶段的性能升级方案

## 4. 功能需求

### 4.1 功能总览表

| # | 模块 | 功能描述 |
|---|---|---|
| 1 | 资料上传 | 用户在首页上传 PDF/TXT/MD 文件。系统接收文件后展示上传进度、解析进度、世界生成进度三段式提示。失败时提供原因（文件过大/格式不支持/解析失败）和重试入口。生成成功后自动跳转到世界地图页。 |
| 2 | 世界生成引擎 | 后端解析文件文本，抽取不超过 20 个核心知识节点。每个节点不再只归属某一层，而是同时生成 What / How / Why / System 四个深度内容，并附带谜题标题、关卡 NPC 信息、相邻节点、原文依据、What 三张翻卡、老学者引导文案和原问回响状态。 |
| 3 | 四深度像素地图 | 渲染 What / How / Why / System 四张独立地图。四张地图使用不同生物群系背景，但节点集合和坐标保持一致。用户通过深度切换器在地图间切换，向下自由，向上受门禁限制。 |
| 4 | 节点探索交互 | 用户点击当前深度已解锁的节点后，学者化身按路径移动到关卡 NPC 旁边。关卡 NPC 只展示谜题标题；What 翻卡、How / Why / System 问答与反馈都由老学者承担。地图上默认显示谜题标题，而不是直接显示节点名。 |
| 5 | 关卡小场景引入 | 用户首次进入某节点 What 层时，播放 5-8 秒像素小场景，展示一个与知识点相关的视觉矛盾。小场景结束后关卡 NPC 提出 `mysteryQuestion`，老学者接管进入 What 翻卡；之后再次访问同一节点不自动播放。 |
| 6 | What 翻卡阅读 | 每个节点的 What 内容拆为三张卡：定义、例子、连接，由老学者展示。用户逐张阅读后点击"收入图鉴，继续探索"，系统将该节点 What 深度标记完成，不要求打字回答。 |
| 7 | 高阶问答引擎 | How / Why / System 复用老学者 DialogBox 问答。How 侧重机制与过程，Why 侧重因果与反事实，System 侧重跨节点、跨领域迁移和反例讨论。 |
| 8 | 诊断反馈与追问 | 用户在高阶层输入回答（≤500 字）。AI 输出四段式反馈卡：已理解、还缺一点、提示、下一问。最多 3 轮后给出当前深度的完成判断。 |
| 9 | 迷雾消散与解锁 | 未在当前深度解锁的节点完全隐藏在迷雾下。完成一个节点后，迷雾从当前节点向邻居扩散消散，露出邻居节点、谜题标题和关卡 NPC。 |
| 10 | Cognitive Progress Model | 每个节点记录四个深度的进度状态。HUD 不显示考试感的"已掌握 X / 共 Y 节点"，改为显示"认知迷雾已消散 X%"，用世界清晰度表达学习推进。 |
| 11 | 原问回响 | 单个节点四层完成后，老学者引导用户回到 What 地图；关卡 NPC 再次提出最初的 `mysteryQuestion`。用户通过后，节点进入完整通关状态并显示完整光晕。 |
| 12 | 区域征服徽章 | 当一个关键区域或一组节点在指定深度完成后，触发徽章反馈；当单个节点完成原问回响后，计入区域徽章进度。 |
| 13 | 世界列表 | 顶部导航提供"我的世界"入口，列出用户已生成的所有世界（缩略图 + 名称 + 进度）。可切换、可删除。 |

### 4.2 模块详情

#### 4.2.1 资料上传（# 1）

**原型示意**

```
+------------------------------------------------------+
|  Knowledge World                            [我的世界]|
+------------------------------------------------------+
|                                                       |
|        把任何资料，变成一个可探索的世界                |
|                                                       |
|       +---------------------------------------+       |
|       |       拖拽 PDF/TXT/MD 到这里           |       |
|       |             或点击选择                 |       |
|       +---------------------------------------+       |
|                                                       |
|       支持：PDF / TXT / Markdown，≤ 20 MB             |
|                                                       |
|       [试试预制世界：人类简史 / 经济学原理]            |
|                                                       |
+------------------------------------------------------+
```

- **业务逻辑**：文件上传 → 后端校验大小与格式 → 文本抽取 → 异步生成世界 → 推送进度事件 → 完成后跳转。
- **交互逻辑**：上传过程显示三段式进度（上传 / 解析 / 生成），每段独立百分比；任一段失败弹出错误卡片，提供"重试"和"换一个文件"。
- **规则约束**：
  - 文件大小：≤ 20 MB
  - 格式白名单：`.pdf` / `.txt` / `.md`
  - 文件名长度：≤ 100 字符
  - 同一用户并发生成任务：≤ 1（防止 LLM 调用爆炸）
- **预制世界**：MVP 至少预置 2 个生成好的世界（《人类简史》《经济学原理（曼昆）第一章》），首屏即可点击试玩，绕开等待期流失。

#### 4.2.2 世界生成引擎（# 2）

**输入**：解析后的纯文本（按章节切块）。
**输出**：JSON 格式的世界结构。

**世界结构 JSON 示例**：

```json
{
  "world_id": "w_2026_xxx",
  "title": "人类简史",
  "biome_theme": "history-civilization",
  "start_node_id": "n_001",
  "nodes": [
    {
      "id": "n_001",
      "name": "认知革命",
      "mysteryQuestion": "是什么让智人和其他动物不同？",
      "gateNpc": {
        "id": "story_elder",
        "title": "讲故事的人",
        "avatar": "/assets/characters/gate_npc_story_elder.png"
      },
      "position": { "x": 280, "y": 520 },
      "neighbors": ["n_002", "n_003"],
      "source_excerpt": "约七万年前，智人开始...",
      "introScene": {
        "sceneText": "洞穴壁画前，几个人围着火堆听同一个故事。",
        "visualHint": "cave_fire_storytelling",
        "durationSec": 6,
        "trigger": "first_enter_what",
        "state": "unseen"
      },
      "whatCards": [
        { "type": "definition", "text": "认知革命指智人出现更复杂语言和虚构能力的阶段。" },
        { "type": "example", "text": "部落神话、图腾、法律、公司都依赖共同相信的故事。" },
        { "type": "bridge", "text": "这种能力让陌生人也能围绕同一套想象秩序协作。" }
      ],
      "mentorPrompts": {
        "whatIntro": "先别急着回答。我们先看看这个问题由哪些事实组成。",
        "how": "认知革命是如何一步步扩大人类协作规模的？",
        "why": "为什么虚构故事比体力更能解释智人的胜出？",
        "system": "现代公司、国家和货币与认知革命有什么共同底层逻辑？",
        "finalReturn": "你已经从四个角度看过它了。现在回到最初的问题。"
      },
      "finalQuestion": {
        "source": "mysteryQuestion",
        "state": "locked"
      }
    }
  ],
  "layers": ["what", "how", "why", "system"]
}
```

- **业务逻辑**：
  1. 文本切块（每块 ≤ 1500 token，重叠 200 token）
  2. LLM 抽取候选概念（每块输出 3-5 个）
  3. 概念去重 + 聚类（向量相似度 + LLM 二次合并）
  4. 生成统一节点集合，不再把节点硬分配到某个层级
  5. 为每个节点生成谜题标题、关卡 NPC、首次引入小场景、What 三张卡、老学者四层引导文案和原问回响状态
  6. 节点关系生成（依据原文邻近度与 LLM 语义判断）
  7. 节点总数硬上限 20，超出时按重要性裁剪
- **规则约束**：
  - 节点总数：≤ 20
  - 四张深度地图共享同一批节点和坐标
  - 节点名长度：≤ 12 字符（保证地图可读性）
  - 谜题标题长度：≤ 18 字，优先使用能激发好奇心的问题
  - 每个节点必须有 `mysteryQuestion`、`gateNpc`、`introScene`、`whatCards`、`mentorPrompts`、`finalQuestion`、`neighbors`
- **异常处理**：
  - LLM 调用失败 3 次 → 任务标记失败，用户可重试
  - 文本过短（<500 字）→ 拒绝生成，提示"资料过短"

#### 4.2.3 像素风世界地图（# 3）

**原型示意**：直接复用 `assets/world_map_sapiens.jpg` 作为《人类简史》世界地图视觉基线；四张深度地图使用 `assets/biomes/` 中的对应背景全屏切换。

- **业务逻辑**：前端拿到世界 JSON 后，使用同一批节点坐标渲染当前深度地图。`currentDepth` 决定背景图、节点可见状态、老学者引导内容和完成状态。
- **交互逻辑**：
  - 深度切换器显示 What / How / Why / System 四个入口
  - 向下层级可随时切换；向上层级需要满足门禁条件
  - 当前深度未解锁的节点完全隐藏在迷雾下
  - 点击可见节点 → 学者化身移动到关卡 NPC 旁边 → 关卡 NPC 展示谜题 → 老学者弹出对应深度交互
  - 地图上显示谜题标题，不直接显示节点名
- **规则约束**：
  - 地图分辨率：1366 × 768 基线，≥ 1920px 自动放大
  - 视图：固定俯视 3/4 视角，不支持自由缩放（MVP 阶段）
  - 像素风格基线：暖黄（What 层）→ 金绿（How 层）→ 石灰（Why 层）→ 紫蓝（System 层）
  - MVP 渲染方案：Canvas 2D + 四张深度背景图预加载；深度切换动画目标耗时 ≤ 500ms
- **门禁规则**：
  - 进入 How：至少 1 个节点完成 What
  - 进入 Why：至少 1 个节点完成 How
  - 进入 System：至少 1 个节点完成 Why
  - 同一深度内，startNode 自动可见；完成一个节点后露出当前深度的邻居节点

**深度切换器 UI**

地图右上角常驻一个垂直深度切换器，是用户在四张深度地图间切换的唯一入口，也承载"当前在哪一层、还能去哪一层"的可视化表达。

```
+----------------+
|  ◉  What       |   ← 当前所在深度，高亮 + 已完成节点数
|  ●  How        |   ← 可进入：高亮可点击
|  ○  Why        |   ← 锁定：灰色 + 锁形图标 + 解锁条件 tooltip
|  ○  System     |   ← 锁定
+----------------+
```

| 状态 | 视觉 | 触发条件 | 交互 |
|---|---|---|---|
| 当前深度 | 实心圆点 + 高亮描边 + 当前层节点完成数 | 用户当前所在深度 | 不可点击自身 |
| 可进入 | 实心圆点 + 标准描边 | 满足向上门禁条件，或任意更浅深度 | 点击切换地图，触发深度切换动画 |
| 已完成 | 圆点上叠加完成光晕 | 该深度所有节点均已 `completed` | 点击进入复习态 |
| 锁定 | 空心圆 + 锁形图标 | 不满足门禁 | 悬停显示一句解锁条件文案，例如"先在 What 地图完成至少 1 个节点" |

切换动画：背景色从当前层过渡到目标层（暖黄 / 金绿 / 石灰 / 紫蓝）；节点坐标不变；已完成节点保持完成光晕；新深度下的可见状态根据"依赖解锁"重新计算。

#### 4.2.4 关卡小场景引入（# 5）

小场景负责把抽象知识点变成一个具体情境。它不讲解知识、不提供答案，只展示一个与当前知识点相关的视觉矛盾，让关卡 NPC 的谜题从场景里自然出现。

- **触发条件**：用户首次进入某节点的 What 层，且 `introScene.state = "unseen"`。
- **播放时长**：5-8 秒，MVP 以短动画或静态像素舞台 + 轻动效实现。
- **标准流程**：
  1. 学者化身走到关卡 NPC 旁。
  2. 播放 `introScene.sceneText` 对应的小场景。
  3. 关卡 NPC 提出 `mysteryQuestion`。
  4. 老学者使用 `mentorPrompts.whatIntro` 接管，进入 What 翻卡。
  5. 播放完成后将 `introScene.state` 标记为 `seen`。
- **回访规则**：
  - 再次访问同一节点时，不自动播放小场景，直接进入当前深度对应交互。
  - 原问回响阶段不重播小场景，只由关卡 NPC 再次提出同一个 `mysteryQuestion`。
  - 手动"重看场景"入口可作为 P2，不作为 MVP 必需项。

《人类简史》P0 阶段至少制作 3 个小场景：认知革命（洞穴火堆讲故事）、农业革命（粮仓满但农夫疲惫）、货币（陌生人在市场用金币完成交换）。

#### 4.2.5 What 翻卡阅读（# 6）

What 层负责建立事实和概念，不要求用户回答。关卡 NPC 只展示 `mysteryQuestion`，三张卡由老学者展示：

| 卡片 | 内容目的 | 示例 |
|---|---|---|
| 定义 | 讲清"这是什么" | 认知革命指智人出现更复杂语言和虚构能力的阶段 |
| 例子 | 给出具体画面 | 部落神话、图腾、公司、货币都是共同相信的故事 |
| 连接 | 引出后续问题 | 共同故事让陌生人协作规模变大 |

- **交互逻辑**：点击节点 → 学者移动到关卡 NPC 旁 → 关卡 NPC 说出 `mysteryQuestion` → 老学者接管，对话框进入 `reading` 模式 → 用户逐张翻卡 → 点击"收入图鉴，继续探索" → 当前节点 What 深度完成。
- **阅读节流**：
  - 每张卡首次展示后，"下一张"按钮延迟 2 秒可点击。
  - 第三张卡首次展示后，"收入图鉴，继续探索"按钮延迟 2 秒可点击。
  - 延迟期间按钮显示轻量提示："读完这张卡，再继续探索"。
  - 该规则只用于防止误触和三连点跳过，不作为理解判断依据。
- **完成效果**：当前节点亮起 What 标记；迷雾向邻居扩散消散；若这是用户第一个 What 完成节点，How 地图入口解锁。
- **边界规则**：What 层不判断对错，不要求输入文字，不调用诊断反馈模型。系统可以记录用户是否看完三张卡和每张卡是否达到最小阅读时间，但不把停留时间作为理解判断依据。

#### 4.2.6 高阶问答引擎（# 7）与诊断式反馈（# 8）

> 这是产品的认知核心，也是最容易翻车的地方，必须最严格设计。

核心机制定义为 **AI Cognitive Engine**：

```text
AI 不是聊天，而是在做三件事：
1. 判断用户处于哪个认知层级
2. 找出当前最大理解误区
3. 生成下一步最优问题
```

这个机制让产品区别于普通 ChatGPT 对话。ChatGPT 主要响应用户问题，而 Knowledge World 主动维护一条 Socratic Path：系统知道当前节点要训练什么、用户卡在哪里，以及下一问应该把用户推向哪个理解层级。

核心交互原则为：

> 关卡 NPC 只提出原始谜题；What 由老学者给材料，How / Why / System 由老学者提问；先诊断，再讲解；先让用户思考，再给答案。

老学者在高阶层不应一开始灌输结论。当用户明显不知道、回答严重偏离或连续两轮卡住时，需要给出最小必要讲解。讲解必须服务于下一次思考，而不是替用户完成理解。

- **提问生成 Prompt 骨架**：
  ```
  你是贯穿全程的老学者。学习者刚走到节点【{node_name}】。
  当前关卡 NPC 只负责提出原始谜题和最终终问，不参与中间教学。
  原文片段：{source_excerpt}
  当前深度是【{depth}】（How/Why/System）。
  请生成 1 个核心提问 + 2 个追问。
  规则：
  - How 问机制、步骤、过程
  - Why 问原因、反事实、底层条件
  - System 问跨节点连接、现实迁移、反例
  - 第一问不直接给结论
  - 不暴露原文答案
  - 不提供选项
  ```
- **诊断反馈 Prompt 骨架**：
  ```
  原文要点：{source_key_points}
  学习者回答：{user_answer}
  当前深度：{depth}
  当前轮次：{round}
  请输出 JSON：
  {
    "depth": "how" | "why" | "system",
    "covered_dimensions": ["concept" | "logic" | "transfer"],
    "main_misconception": "当前最大的理解误区",
    "missing_points": ["缺失要点 1", "缺失要点 2"],
    "next_best_question": "下一步最优问题",
    "feedback_level": "reinforce" | "hint" | "minimal_explain",
    "feedback_card": {
      "understood": ["用户已经理解的点"],
      "missing": ["还缺的关键点"],
      "guidance": "根据 feedback_level 输出强化反馈、提示或最小必要讲解",
      "next_question": "使用 next_best_question 继续推进理解"
    },
    "depth_state": "learning" | "completed"
  }
  ```
- **业务逻辑**：每个节点在每个高阶深度最多 3 轮交互。
  - 回答覆盖当前深度核心要点：当前深度标记为 `completed`，触发迷雾消散
  - 回答模糊：输出提示与下一问，当前深度保持 `learning`
  - 用户完全不知道：输出最小必要讲解，再给一个更低门槛追问
  - 连续 3 轮未完成：保留 `learning` 状态，允许用户稍后返回
- **稍后返回规则**：
  - 用户离开后再次进入同一节点、同一深度时，不进入第 4 轮，而是开启新的 3 轮尝试周期。
  - 历史回答、上一轮反馈卡和最大误区会保留在该节点的"上次尝试"记录中，老学者开场时用一句话提醒："上次我们卡在这里，这次换个角度试试。"
  - 新周期的 `round` 从 1 开始，诊断模型可读取历史记录作为上下文，但 UI 不显示"第 4 轮"。
  - 若用户在新周期中完成当前深度，则 `depth_state` 变为 `completed`；否则继续保留 `learning`。
- **规则约束**：
  - 用户回答长度：≤ 500 字
  - 单次诊断超时：5 秒，超时回退为"理解中 + 留追问"
  - LLM 输出非合法 JSON：重试 1 次，仍失败则按 `learning` 处理（保护用户体验）
  - 最小必要讲解长度：≤ 120 字，必须包含一个后续追问

**反馈卡样式示例**：

```
理解状态：接近掌握

你已经理解：
- 农业革命改变了人类获取食物的方式
- 定居生活让人口增长成为可能

还缺一点：
- 农业革命也带来了劳动强度上升、社会等级和财产观念

提示：
当粮食可以储存后，谁来分配粮食、谁拥有土地、谁承担劳动，都会变成新的社会问题。

追问：
如果农业革命让粮食更多，为什么作者反而说它未必让个体更幸福？
```

#### 4.2.7 Cognitive Progress Model（# 10）

学习状态系统让 MVP 从"演示效果"变成"可持续学习系统"。每个节点不再只有解锁/未解锁两种状态，而是记录四个深度各自是否完成。

| 字段 | 含义 | 示例 |
|---|---|---|
| `currentDepth` | 当前所在认知深度 | `what` |
| `progress[nodeId].introScene` | 小场景观看状态 | `unseen` / `seen` |
| `progress[nodeId].what` | 节点 What 深度状态 | `locked` / `available` / `completed` |
| `progress[nodeId].how` | 节点 How 深度状态 | `locked` / `available` / `learning` / `completed` |
| `progress[nodeId].why` | 节点 Why 深度状态 | `locked` / `available` / `learning` / `completed` |
| `progress[nodeId].system` | 节点 System 深度状态 | `locked` / `available` / `learning` / `completed` |
| `progress[nodeId].finalQuestion` | 原问回响状态 | `locked` / `available` / `completed` |
| `progress[nodeId].nodeClear` | 节点完整通关状态 | `false` / `true` |

单个深度的状态流转如下：

```text
locked → available → learning → completed
             ↘ What 层翻卡完成 ↗
```

当 `what`、`how`、`why`、`system` 全部为 `completed` 时，`finalQuestion` 从 `locked` 变为 `available`。用户完成原问回响后，`finalQuestion` 变为 `completed`，`nodeClear` 变为 `true`，该节点才进入完整通关状态。

产品侧不向用户展示"已掌握 X / 共 Y 节点"这类考试感文案，而是显示"认知迷雾已消散 X%"。这个百分比来自当前世界所有节点、所有深度的完成情况，但用户感知上是世界变清晰，而不是分数变高。

#### 4.2.8 迷雾消散动画（# 9）

> 这是路演 Demo 视觉冲击力的最关键节点，必须做好。

- **动画分镜**：
  1. T0：节点中心闪光（白光，0.3 秒）
  2. T0+0.3s：光圈以节点为中心向外辐射（1 秒，半径覆盖到邻接节点）
  3. T0+0.5s：当前深度迷雾粒子向外扩散并淡出（1 秒）
  4. T0+1.5s：邻接节点、关卡 NPC 和谜题标题从迷雾中浮现（弹性缩放动画 0.4 秒）
  5. T0+2s：HUD 的"认知迷雾已消散"数值轻微跳动
- **技术实现要点**：MVP 使用 Canvas 2D 渲染地图、迷雾和节点；四张深度背景图在进入世界时预加载。迷雾以粒子贴图驱动，邻接节点用 CSS 关键帧 / GSAP；复赛阶段如节点和粒子数量上升，再评估切换 PixiJS。

#### 4.2.9 原问回响（# 11）

原问回响是单个节点的完整通关机制。它不引入新问题，而是让用户带着四层理解回到最初看到的同一个 `mysteryQuestion`。

- **触发条件**：同一节点的 `what`、`how`、`why`、`system` 四个深度均为 `completed`。
- **流程**：
  1. System 深度完成后，老学者弹出 `mentorPrompts.finalReturn`。
  2. 系统提示用户可选择"现在回答"或"稍后回来"。
  3. 用户选择"现在回答"时，地图自动切回 What 层，并聚焦该节点的关卡 NPC。
  4. 关卡 NPC 再次提出原始 `mysteryQuestion`，不得生成新的终问。
  5. 用户用自己的话回答；老学者在后台执行综合诊断，判断回答是否覆盖概念准确、机制完整、原因解释、迁移意识四个维度。
  6. 通过后，关卡 NPC 给出一句简短认可；`finalQuestion` 变为 `completed`，`nodeClear` 变为 `true`，节点显示完整光晕。
- **边界规则**：
  - 关卡 NPC 只说原始谜题和最终认可，不输出解释、提示或追问。
  - 终问文本必须等于该节点的 `mysteryQuestion`。
  - 用户选择"稍后回来"时，节点在 What 地图显示终问可答标记，不打断用户继续探索。
  - 初赛 MVP 可先用固定评分规则判断是否通过；复赛阶段再接入更细的 AI 诊断反馈。

## 6. 风险与降级方案

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| LLM 评估不一致（用户答对被判错） | 高 | 高 | 引入"通过 + 留追问"软判定，避免硬性拒绝；提供"我觉得我答对了"申诉按钮（MVP 后接入） |
| AI 反馈过度，重新变成讲课工具 | 中 | 高 | What 层只翻卡不诊断；How / Why / System 限制最小讲解长度 ≤120 字；每次讲解后必须接追问 |
| 生成耗时过长（>2 分钟） | 中 | 高 | 预制世界兜底；生成期展示"探险家在绘制地图"动画掩盖等待 |
| 用户不理解四张地图关系 | 中 | 高 | 节点坐标保持不变；深度切换器标注锁定/可进入/已完成；首次切换时用一句话说明"同一个节点正在进入更深理解" |
| 用户不知道怎么切换到下一深度 | 中 | 低 | 深度切换器常驻显示；新手引导第一步指向切换器，并在完成第一个 What 节点后演示一次进入 How 地图 |
| 小场景重复播放造成打扰 | 中 | 中 | 仅当 `introScene.state = "unseen"` 时自动播放；播放完成立即标记为 `seen`；回访和原问回响均不自动重播 |
| 像素地图布局重叠 | 中 | 中 | 节点数硬上限 20；预制世界手工校准坐标；自动生成世界使用力导向算法 + 重叠检测重排 |
| LLM 配额不足或超时 | 中 | 中 | MVP 用 OpenAI API，复赛前评估切换火山方舟/DeepSeek；后端封装 LLMAdapter 接口，切换模型只改配置；预备双通道降级 |
| 评委 30 秒内 get 不到产品价值 | 中 | 极高 | Demo 视频前 10 秒必须出现"迷雾消散"镜头 |

## 8. 验收标准

MVP 必须同时满足以下条件方可视为可参赛：

- 《人类简史》预制世界开箱可玩，至少包含 6 个节点，每个节点具备 What / How / Why / System 四个深度内容，且每个节点同时具备 `mysteryQuestion`、`gateNpc`、`introScene`、`whatCards`、`mentorPrompts`、`finalQuestion` 字段
- 地图上节点默认显示谜题标题与关卡 NPC 头像，不显示节点名；首次点击进入 What 层时先播放小场景，再由关卡 NPC 展示原始谜题，随后老学者进入教学过程
- 至少 3 个 P0 节点具备 5-8 秒小场景引入；同一节点小场景播放后 `introScene.state` 变为 `seen`，再次访问该节点不再自动播放
- What 地图中，用户能完成至少 3 个节点的三张翻卡阅读，并看到迷雾露出邻居节点
- What 翻卡必须具备最小阅读节流：每张卡首次展示后 2 秒内不能点击下一张，第三张卡展示后 2 秒内不能点击"收入图鉴，继续探索"
- 深度切换器能正确表达已完成、可进入、锁定三种状态；锁定状态悬停可看到解锁条件文案
- 完成 ≥ 1 个 What 节点后能成功进入 How 地图，并看到背景从暖黄过渡到金绿、节点坐标保持不变
- 向下切换地图时进度不丢失：在 How 地图退回 What 地图后，已完成的 What 节点仍显示完成态
- 单个会话内可完成至少 1 条"How 问题→用户回答→AI 诊断反馈→当前深度完成→迷雾消散"完整链路
- 同一节点同一高阶深度连续 3 轮未完成后，返回时必须开启新的 3 轮尝试周期；历史反馈可作为上下文保留，但 UI 不显示"第 4 轮"
- 单个节点四层完成后，能触发原问回响：老学者引导返回 What 地图，关卡 NPC 再次提出完全相同的 `mysteryQuestion`，用户通过后节点进入完整通关状态
- 用户完全不知道时，系统能输出 ≤120 字的最小必要讲解，并继续给出一个追问
- HUD 文案为"认知迷雾已消散 X%"，不出现"已掌握 X / 共 Y 节点"等考试感表述
- 迷雾消散动画在 1080p 下流畅播放（≥30 帧）
- 评委盲测：3 名非技术人员看完 demo 视频后能独立复述"四张深度地图"、"知识迷雾"和"学者化身探索"三个核心机制

## 10. LLM 选型策略（补充）

MVP 阶段 LLM 选型采用"先跑通再优化"策略：

| 阶段 | 模型 | 用途 | 理由 |
|---|---|---|---|
| MVP 验证（06-16 ~ 07-06） | OpenAI GPT-4o | 世界生成 + 提问 + 诊断反馈 | 效果最稳定，Prompt 调试效率最高 |
| 初赛提交前（07-06 ~ 07-14） | 评估后决定 | 可能切换到火山方舟（豆包）或 DeepSeek | 降低成本 + 国内访问稳定性 |
| 复赛阶段（07-20 ~ 08-10） | 待定 | 根据初赛反馈 + 评委建议调整 | 可能需要多模型协作（生成用大模型，评估用小模型） |

架构要求：后端封装统一的 `LLMAdapter` 接口，所有 LLM 调用通过适配器，切换模型只需改配置，不动业务代码。
