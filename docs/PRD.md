文档创建日期：2026-06-18（修订版）
项目代号：Knowledge World

## 0. 初赛版本范围（参赛 Demo 优先级）

为了让初赛 Demo 在 30 秒内击中评委，本节明确锁死本版本要做、可选做、不做的范围。开发期间一切优先级冲突以本节为准。

### 0.1 初赛必须完成（P0，决定产品成立）

- 至少 1 个预制世界（《人类简史》）完整跑通 What / How / Why / System 四张深度地图
- 同一批知识节点在四张地图中保持坐标不变，用户能看到同一个知识点被逐层理解
- 节点不显示节点名，显示关卡 NPC 头像 + 一句谜题标题，驱动用户点击下一个节点
- 关卡 NPC 只承担两件事：初次展示原始谜题；四层完成后再次提出同一个原始谜题作为最终终问
- 关卡首次引入使用 5-8 秒小场景：展示一个与知识点强相关的视觉矛盾，由关卡 NPC 提出 `mysteryQuestion`
- 小场景结束后，进入学者-老学者引导对话阶段，学者代表玩家提问，老学者引导性回应
- 对话结束后，老学者引出三张知识卷轴，在对话框上方逐一水平展开，对话框同步显示老学者对当前卷轴的讲解
- 三张卷轴阅读完毕后，学者-老学者简短收尾对话，然后进入"选择关键卷轴"轻量确认
- 小场景只在用户首次进入某节点 What 层时自动播放；之后再次访问同一节点不自动播放
- P0 阶段至少覆盖《人类简史》3 个核心节点的小场景：认知革命、农业革命、货币；其余 4 个小场景写入设计但可暂缓实现
- 老学者是全程导师 NPC，负责引导对话、卷轴讲解、How / Why / System 提问、诊断反馈、层级切换引导和终问前的回到原点提示
- What 层采用老学者引导对话 + 三卷轴展开阅读（定义、例子、连接），不做答题
- How / Why / System 层由老学者提问：用户输入回答，AI 输出诊断反馈
- 节点点击 → 学者化身像素小人移动到关卡 NPC 旁 → 小场景（首次）→ 学者-老学者对话 → 卷轴展开 → 收尾 → 选择关键卷轴
- 节点四层完成后，触发"原问回响"：系统引导用户回到 What 地图，由关卡 NPC 再次提出同一个 `mysteryQuestion`
- 迷雾结构性隐藏节点：未解锁节点不可见，完成节点后迷雾消散并只露出 1 个相邻谜题
- 首次进入新深度（How/Why/System）时，自动弹出四卷轴竖排悬挂的深度选择界面，搭配老学者引导对话。之后可通过 HUD 快速切换

### 0.2 初赛尽量完成（P1，加分项，做不完不影响演示）

- PDF 上传 → 自动生成世界（路演时作为"能力展示"短镜头，不作为主链路入口）
- 第二个预制世界（《经济学原理》第一章）
- 区域征服徽章
- 完成《人类简史》其余 4 个可暂缓小场景：虚构故事、想象的秩序、帝国、科学革命

### 0.3 初赛不展示或弱化（P2，留到复赛）

- 世界列表 / 多世界切换
- 完整埋点体系（初赛只保留 `intro_scene_viewed`、`what_card_completed`、`depth_switched`、`depth_completed`、`final_question_completed` 与 `session_complete`）
- 长期学习档案、积分、成就系统
- 申诉按钮、多模型协作

> 初赛阶段所有"上传任意 PDF 自动生成"的演示都从预制世界进入，避免现场生成等待与质量不可控带来的风险。

## 1. 需求背景

### 1.1 触发来源

AI 学习工具在 2024-2026 年迎来爆发，但绝大多数产品只解决了"信息获取"问题——总结、问答、生成课程——本质仍是把 AI 当作更聪明的"讲解员"。学习者拿到的依然是被动消费的内容，而非主动构建的理解。

### 1.2 用户当前现状与痛点

| 痛点 | 表现 | 后果 |
|---|---|---|
| 内容过载 | 收藏夹里堆满 PDF 与课程，真正读完的不到 10% | 知识焦虑，越学越虚 |
| 被动消费 | 读完一本书或听完一门课，能复述但不会用 | 理解停留在 What 层 |
| 缺乏反馈 | 不知道自己哪里没懂，也没人追问 | 自我评估失真 |
| 学习无趣 | 严肃学习场景缺乏正反馈机制 | 难以坚持 |

### 1.3 不解决的后果

如果 AI 学习产品继续停留在"总结/讲解"形态，用户会快速对 AI 学习工具产生疲劳，市场窗口被 ChatGPT/Claude 等通用对话产品吃掉。差异化窗口正在快速关闭。

### 1.4 为什么现在可行

大模型已经具备稳定的语义理解、结构抽取、问题生成和反馈生成能力，学习资料可以从"静态内容"转化为"动态认知交互系统"。过去的知识图谱只能组织信息，现在的 AI 可以判断用户的理解层级、定位误区，并持续生成下一步问题，这让探索式学习从界面创意变成可落地的产品机制。

## 2. 产品目标

### 2.1 核心目标

> 让学习像探索未知世界一样发生。

用户把一份学习资料转化为可导航的知识世界。系统先用 What 引导对话 + 卷轴阅读帮助用户收集事实与概念，再用 How / Why / System 问答推动用户解释机制、追问原因，并把知识迁移到更大的系统里。

### 2.2 产品定位

Knowledge World 的定位不是 AI 学习工具，而是认知探索系统：

```
Knowledge → World → Exploration → Cognition Training
```

它关注的不是"把知识讲得更快"，而是帮助用户看见自己的理解边界，并通过一次次探索把边界向外推进。

### 2.3 核心玩法模型

Knowledge World 用四张独立地图表达认知深度：

| 深度 | 地图主题 | 交互方式 | 视觉调性 |
|---|---|---|---|
| What | 知道它是什么 | 引导对话 + 卷轴展开阅读 + 轻量确认 | 暖黄（史前洞穴营地） |
| How | 理解它如何运作 | 老学者提问 → 用户自由回答 → AI 诊断 | 金绿（田园农场） |
| Why | 追问它为什么成立 | 因果追问 + 反事实假设 | 石灰/大地色（古希腊废墟） |
| System | 迁移到更大的系统 | 跨领域连接 + 现实问题迁移 | 紫蓝（科幻研究岛） |

同一批节点会出现在四张地图上，坐标保持不变。用户每切换一次深度，看到的不是另一批知识，而是同一个知识世界变得更深。

### 2.4 三条核心规则

| 规则 | 定义 | 用户感受 |
|---|---|---|
| 向下自由 | 任意时刻都可以从当前深度切换到任意更浅的深度，无惩罚、无门槛，已完成节点保持完成状态 | 想复习随时回去 |
| 向上门禁 | 进入 How 需要 ≥ 1 节点完成 What；进入 Why 需要 ≥ 1 节点完成 How；进入 System 需要 ≥ 1 节点完成 Why | 必须先建立基础再深入 |
| 依赖解锁 | 在深度 D 下，每次完成一个节点后，只从它的相邻未发现节点中显露 1 个下一节点；每张深度地图首次进入时，起始节点在该深度自动可用 | 迷雾不是一次打开一片，而是沿理解路径一步步散开 |

### 2.5 双 NPC 职责边界

| 角色 | 负责 | 不负责 |
|---|---|---|
| 关卡 NPC | 地图上的节点形象、谜题标题、最终终问、通关认可 | What 引导对话、卷轴讲解、How / Why / System 提问、诊断反馈、层级切换说明 |
| 老学者 | What 阶段引导对话、卷轴讲解与收尾、How / Why / System 提问、AI 诊断反馈、深度层级引导、终问前回到原点提示 | 代替关卡 NPC 提出原始谜题 |

### 2.6 量化目标（参赛 MVP 阶段）

| 维度 | 目标 |
|---|---|
| Demo 演示效果 | 进入预制世界后 10 秒内完成首次节点交互（学者移动 → 小场景 → 对话 → 卷轴阅读） |
| 核心交互完成率 | 测试用户首次会话内能完成 ≥1 个节点的 What 层（含对话+卷轴+选择确认），并进入 How 问答 |
| 视觉记忆点 | 路演评委观看 30 秒后能复述"四张深度地图""知识卷轴"和"知识迷雾"三个机制 |
| 反馈可信度 | How / Why / System 中至少 1 层能跑通 AI 追问 → 用户回答 → 诊断反馈 → 解锁判断 |

### 2.7 非目标（明确不做）

- 不做登录系统（MVP 阶段匿名使用，用户数据存 localStorage）
- 不做长期用户成长体系（积分、段位、皮肤等）
- 不做社交分享
- 不做多人协作探索
- 不做移动端适配（仅 Web 桌面端）

## 3. 用户与场景

### 3.1 目标用户画像

| 画像 | 描述 | 决策权重 |
|---|---|---|
| 求知型自学者 | 有学习目标但缺方法的成年人，例如刚入门读《人类简史》的读者 | 主要 |
| 应试备考者 | 需要构建知识体系的考研/考证用户 | 次要 |
| 教育评委 / 比赛评审 | 关心创新性与产品完整度 | MVP 阶段优先服务 |

### 3.2 核心场景排序

| 优先级 | 场景 | 描述 |
|---|---|---|
| P0 | 进入预制世界 | 用户从首页进入《人类简史》预制世界，落点是 What 地图，看到起始节点的关卡 NPC 和谜题标题，周围被结构性迷雾覆盖 |
| P0 | 关卡小场景引入 | 用户首次进入某节点 What 层时，播放 5-8 秒像素小场景，用视觉矛盾引出关卡 NPC 的原始谜题；之后再次访问同一节点不再自动播放 |
| P0 | What 引导对话 + 卷轴阅读 | 小场景结束后，学者与老学者进行引导对话（5句），老学者引出三张卷轴在对话框上方逐一展开，对话框同步讲解，最后简短收尾后用户选择关键卷轴 |
| P0 | 深度入口仪式 | 完成第 1 个 What 节点后，自动弹出四卷轴竖排悬挂界面，展示 What(已通关)+How(可进入)+Why/System(锁定)，伴随引导对话；首次之后使用 HUD 快速切换 |
| P0 | 深度切换 | 深度切换器中的入口从锁定变为可进入；用户切到下一深度地图，背景与色调随之变化，但节点坐标不变 |
| P0 | 高阶问答与反馈 | 用户在 How / Why / System 层回答老学者的问题，AI 诊断理解缺口，给出四段式反馈，并决定当前深度是否完成 |
| P0 | 原问回响 | 单个节点四层完成后，老学者引导用户回到 What 地图；关卡 NPC 再次提出同一个原始谜题，用户通过后节点完整通关 |
| P0 | 向下复习 | 用户在 How / Why / System 层任意时刻可向下切回 What 或更浅深度复习已完成节点，进度不丢失 |
| P1 | 区域征服反馈 | 一个节点四个深度全部完成后出现完整光晕；完成一组关键节点后弹出区域徽章 |
| P2 | 切换世界 | 同一用户已生成多个世界，可在世界列表间切换 |

### 3.3 关键用户旅程：首次进入《人类简史》世界

| 步骤 | 用户动作 | 系统反馈 |
|---|---|---|
| 1 | 从首页点击《人类简史》预制世界 | 进入 What 地图：暖黄色调背景；只显示起始节点（关卡 NPC + 谜题标题）；其余位置全被迷雾覆盖；HUD 显示"认知迷雾已消散 0%";深度切换器入口状态：What=可进入、How/Why/System=锁定 |
| 2 | 点击起始节点 | 学者化身按路径走到关卡 NPC 旁；首次进入播放 5-8 秒小场景 |
| 3 | 小场景结束 | 关卡 NPC 说出原始谜题；进入学者-老学者引导对话（学徒提问，老学者回应，共 5 句） |
| 4 | 对话结束 | 老学者引出三张卷轴，在对话框上方逐一水平展开；对话框同步显示老学者对当前卷轴的讲解，用户点击推进 |
| 5 | 三张卷轴阅读完毕 | 简短收尾对话（学徒感悟 + 老学者引导）；用户从三张卷轴中选择"最关键的一张" |
| 6 | 完成 What 深度 | 迷雾从当前节点向外散开，只显露 1 个相邻节点；首次迷雾散去后老学者提示。此时若为首个完成节点，How 入口解锁 |
| 7 | 继续探索新节点 | 新节点首次进入播放各自小场景；已看过的小场景不再自动播放 |
| 8 | 系统检测到 How 首次解锁 | 自动弹出四卷轴竖排悬挂选择界面：What(已通关)+How(可进入·发光)+Why(锁定)+System(锁定)；底部对话区展示第 1 次引导对话 |
| 9 | 点击 How 卷轴 → 展开 → "好的，出发！" | 深度切换动画：背景从暖黄过渡到金绿，节点坐标不变；之后可通过 HUD 按钮自由切换 |
| 10 | 点击 How 可用节点 | 老学者进入问答模式，给出核心提问；用户输入回答 |
| 11 | 提交回答 | AI 输出四段反馈卡（已理解 / 还缺一点 / 提示 / 下一问）；回答覆盖要点时当前深度完成，迷雾再次扩散 |
| 12 | 完成同一节点的四层 | 老学者提示回到最初问题；系统切回 What 地图；关卡 NPC 再次提出同一个谜题；用户回答后节点完整通关 |

## 4. 功能需求

### 4.1 功能总览表

| # | 模块 | 功能描述 |
|---|---|---|
| 1 | 资料上传 | 用户在首页上传 PDF/TXT/MD 文件。系统接收文件后展示上传进度、解析进度、世界生成进度三段式提示。失败时提供原因（文件过大/格式不支持/解析失败）和重试入口。生成成功后自动跳转到世界地图页。 |
| 2 | 世界生成引擎 | 后端解析文件文本，抽取不超过 20 个核心知识节点。每个节点同步生成 What / How / Why / System 四个深度内容，并附带谜题标题、关卡 NPC 信息、相邻节点、What 引导对话与三张卷轴、老学者四个深度引导文案和原问回响状态。 |
| 3 | 四深度像素地图 | 渲染 What / How / Why / System 四张独立地图。四张地图使用不同生物群系背景（星露谷像素风格），但节点集合和坐标保持一致。用户通过深度切换器在地图间切换，向下自由，向上受门禁限制。 |
| 4 | 节点探索交互 | 用户点击当前深度已解锁的节点后，学者化身按路径移动到关卡 NPC 旁边。首次进入 What 层时先播小场景，然后进入学者-老学者对话 + 卷轴阅读；回访时不重播小场景和对话。How / Why / System 层直接进入问答。 |
| 5 | 关卡小场景引入 | 用户首次进入某节点 What 层时，播放 5-8 秒像素小场景，展示一个与知识点相关的视觉矛盾。小场景结束后关卡 NPC 提出谜题，老学者接管进入引导对话。 |
| 6 | What 引导对话 + 卷轴阅读 | What 层不做答题。小场景结束后进入三阶段体验：① 学者-老学者 5 句引导对话（学徒提问，老学者回应）→ ② 三张卷轴在对话框上方逐一水平展开，对话框显示老学者讲解 → ③ 2 句收尾对话后，用户选择最关键的一张卷轴。 |
| 7 | 深度卷轴选择入口 | 首次解锁深度时，展示四卷轴竖排悬挂选择界面，搭配老学者引导对话。支持点击卷轴展开查看深度介绍，确认后进入新深度。每个深度仅首次解锁时展示一次，后续通过 HUD 按钮快速切换。 |
| 8 | 高阶问答引擎 | How / Why / System 复用老学者问答。How 侧重机制与过程，Why 侧重因果与反事实，System 侧重跨领域迁移。用户自由回答，AI 进行诊断反馈。 |
| 9 | 诊断反馈与追问 | 用户在高阶层输入回答。AI 输出四段式反馈卡：已理解、还缺一点、提示、下一问。最多 3 轮后给出当前深度的完成判断。未完成时保留 learning 状态，用户可稍后返回开启新的 3 轮周期。 |
| 10 | 迷雾消散与解锁 | 未在当前深度解锁的节点完全隐藏在迷雾下。完成一个节点后，迷雾从当前节点向外散开，但只露出 1 个下一节点。 |
| 11 | 认知进度模型 | 每个节点记录四个深度的进度状态。HUD 显示"认知迷雾已消散 X%"，用世界清晰度表达学习推进，不出现考试感表述。 |
| 12 | 原问回响 | 单个节点四层完成后，老学者引导用户回到 What 地图；关卡 NPC 再次提出最初的 `mysteryQuestion`。用户回答通过后节点进入完整通关状态。 |
| 13 | 区域征服徽章 | 当一个关键区域或一组节点在指定深度完成后，触发徽章反馈；当单个节点完成原问回响后，计入区域徽章进度。 |
| 14 | 世界列表 | 顶部导航提供"我的世界"入口，列出用户已生成的所有世界（缩略图 + 名称 + 进度）。可切换、可删除。 |

### 4.2 模块详情

#### 4.2.1 资料上传（# 1）

**业务逻辑**：文件上传 → 后端校验大小与格式 → 文本抽取 → 异步生成世界 → 推送进度事件 → 完成后跳转。
**交互逻辑**：上传过程显示三段式进度（上传 / 解析 / 生成），每段独立百分比；任一段失败弹出错误卡片，提供"重试"和"换一个文件"。
**规则约束**：
- 文件大小：≤ 20 MB
- 格式白名单：`.pdf` / `.txt` / `.md`
- 文件名长度：≤ 100 字符
- 同一用户并发生成任务：≤ 1

**预制世界**：MVP 至少预置 2 个生成好的世界（《人类简史》《经济学原理（曼昆）第一章》），首屏即可点击试玩，绕开等待期流失。

#### 4.2.2 世界生成引擎（# 2）

**输入**：解析后的纯文本（按章节切块）。
**输出**：JSON 格式的世界结构，每个节点包含以下字段：

**节点数据字段说明**：

| 字段 | 类型 | 用途 |
|---|---|---|
| `mysteryQuestion` | string | 地图上代替节点名显示的一句谜题文案，激发点击欲望 |
| `gateNpc` | object | 关卡 NPC：`{ id, title, avatar }`，只负责展示谜题标题和最终终问 |
| `introScene` | object | 首次进入关卡的小场景：`{ sceneText, visualHint, durationSec }`，只在首次进入该节点 What 层时自动播放 |
| `whatDialogue` | array | 学者-老学者引导对话：`[{ speaker: "scholar"|"mentor", text }]`，5 句，在卷轴阅读前播放 |
| `whatScrolls` | array(3) | 三张卷轴数据：`[{ type: "definition"|"example"|"bridge", title, content, mentorVoice }]`，每张含老学者讲解台词 |
| `whatWrapUp` | array | 收尾对话：`[{ speaker: "scholar"|"mentor", text }]`，2 句，卷轴阅读完毕后播放 |
| `nextDiscoveryId` | string | 当前节点完成后优先显露的 1 个相邻节点 |
| `mentorPrompts` | object | 老学者文案：`{ how, why, system, finalReturn }`，用于高阶提问引导和终问前提示 |
| `finalQuestion` | object | 原问回响状态：`{ source: "mysteryQuestion", state }`，终问必须复用 `mysteryQuestion` |

**规则约束**：
- 节点总数：≤ 20
- 四张深度地图共享同一批节点和坐标
- 每个节点必须有完整的 `whatDialogue`、`whatScrolls`、`whatWrapUp`、`mentorPrompts` 等字段
- 节点名长度：≤ 12 字符
- 谜题标题长度：≤ 18 字

**异常处理**：
- 生成失败 3 次 → 任务标记失败，用户可重试
- 文本过短（<500 字）→ 拒绝生成，提示"资料过短"

#### 4.2.3 像素风世界地图（# 3）

**业务逻辑**：前端拿到世界 JSON 后，使用同一批节点坐标渲染当前深度地图。`currentDepth` 决定背景图、节点可见状态、老学者引导内容和完成状态。

**交互逻辑**：
- 深度切换器放置在 HUD 中，显示四个深度入口
- 首次解锁新深度时弹出四卷轴深度选择界面（详 §4.2.7）
- 向下层级可随时切换；向上层级需要满足门禁条件
- 当前深度未解锁的节点完全隐藏在迷雾下
- 点击可见节点 → 学者化身移动到关卡 NPC 旁 → 节点 NPC 展示谜题 → 老学者弹出对应深度交互
- 地图上显示谜题标题，不直接显示节点名

**风格要求**：四张深度地图采用星露谷风格像素画风，3/4 俯视视角，生物群系色调分别为：
- What 层：暖黄色调（草原/洞穴营地）
- How 层：金绿色调（田园农场）
- Why 层：石灰/大地色调（古希腊废墟）
- System 层：紫蓝色调（科幻研究岛）

**门禁规则**：
- 进入 How：至少 1 个节点完成 What
- 进入 Why：至少 1 个节点完成 How
- 进入 System：至少 1 个节点完成 Why
- 同一深度内，起始节点自动可见；完成一个节点后只露出当前深度的 1 个下一节点

#### 4.2.4 节点探索交互（# 4）

**触发条件**：用户点击当前深度中可见、可进入的节点。

**不可点击规则**：
- 迷雾下的节点不显示点击光标，不响应点击事件
- 锁定深度中的节点不响应点击
- 已完成节点可再次点击进入复习态，但不重复播放小场景和引导对话

**What 层完整进入链路**：

```
用户点击节点 → 学者移动到关卡 NPC 旁（~1.5 秒）
     ↓
[首次] 5-8 秒像素小场景 → 关卡 NPC 提出谜题
     ↓
引导对话（学者↔老学者，5 句）
     ↓
三张卷轴在对话框上方逐一展开 + 老学者讲解
     ↓
收尾对话（2 句）
     ↓
选择关键卷轴 → What 深度完成
```

**回访规则**：
- `introScene.state = "seen"` 时不重播小场景

**How / Why / System 层**：不播放小场景和引导对话，直接进入老学者问答。

#### 4.2.5 关卡小场景引入（# 5）

**触发条件**：用户首次进入某节点的 What 层，且 `introScene` 状态为 `unseen`。
**播放时长**：5-8 秒。
**标准流程**：
1. 学者化身走到关卡 NPC 旁
2. 播放 `introScene` 对应的小场景（视觉矛盾）
3. 关卡 NPC 提出 `mysteryQuestion`
4. 老学者接管，进入引导对话阶段
5. 播放完成后将 `introScene` 标记为 `seen`

**《人类简史》预制世界小场景**：

| 节点 | 实现优先级 | 小场景画面 | 关卡 NPC | 谜题 |
|---|---|---|---|---|
| 认知革命 | P0 必做 | 洞穴壁画前，几个人围着火堆听同一个故事 | 讲故事的人 | 是什么让智人和其他动物不同？ |
| 农业革命 | P0 必做 | 粮仓堆满谷物，老农夫仍在田里弯腰劳作 | 老农夫 | 为什么农民比猎人更累？ |
| 货币 | P0 必做 | 市场上两个陌生人用一枚金币完成交换 | 商人 | 为什么一张纸能买到真实食物？ |
| 虚构故事 | P1 | 祭司在篝火旁讲述部落神话 | 祭司 | 为什么不存在的东西能让万人合作？ |
| 想象的秩序 | P1 | 法官、商人、士兵都指向同一块石碑 | 石碑书记 | 为什么国家和公司都像故事？ |
| 帝国 | P1 | 征服者走入城门，身后换上被征服者的服饰 | 皇帝信使 | 为什么征服者也会被文化改变？ |
| 科学革命 | P1 | 观星者擦掉旧答案，在星图旁写下"我不知道" | 观星者 | 为什么承认无知反而带来力量？ |

#### 4.2.6 What 引导对话 + 卷轴阅读（# 6）

What 层负责建立事实和概念，不要求用户回答。整个体验分三个阶段，在同一个对话框界面中完成：

**阶段一：引导对话**

学者（学徒）与老学者进行约 5 句线性对话。学徒代表玩家提问，老学者引导性回应，自然过渡到三张卷轴。对话框上方两侧并排显示学徒和老学者头像，当前说话者高亮，对方变暗。

**阶段二：卷轴阅读**

三张卷轴在对话框上方逐一水平展开（从中心向两侧），每张卷轴包含：

| 类型 | 内容目的 | 示例 |
|---|---|---|
| 定义 | 讲清"这是什么" | 认知革命是智人出现复杂语言和虚构能力的阶段 |
| 例子 | 给出具体画面 | 部落神话、公司、货币都是共同相信的故事 |
| 连接 | 引出后续问题 | 共同故事让陌生人协作规模变大 |

卷轴展开的同时，对话框显示老学者对该卷轴的讲解（逐字打字）。用户点击推进到下一张。三张阅读完毕后自动进入收尾。

**阶段三：收尾对话 + 轻量确认**

2 句简短收尾对话（学徒感悟 + 老学者引导），随后用户从三张卷轴中选择"最关键的一张"。选择后系统记录选择，进入当前节点 What 深度完成状态。

**跳过引导**：用户在对话/卷轴阶段可点击右上角"跳过引导"，直接进入选择关键卷轴。

#### 4.2.7 深度卷轴选择入口（# 7）

**触发时机**：每个深度（How/Why/System）首次解锁时自动弹出一次。后续切换使用 HUD 按钮，不再经过卷轴界面。

**界面形式**：四卷轴竖排悬挂展示在屏幕中央，底部为对话区。

| 卷轴状态 | 视觉表现 | 交互 |
|---|---|---|
| 已完成（What） | 暖黄色，底部有通关印戳 | 可点击查看完成状态，不触发切换 |
| 可进入（当前解锁深度） | 金绿/紫蓝色，边缘呼吸发光动画 | 点击后卷轴水平展开显示深度介绍，底部出现"好的，出发！"按钮 |
| 锁定（未满足门禁） | 灰色，锁链缠绕 | 不可点击，悬停显示解锁条件 |

**三次引导对话**：

| 触发 | 对话主题 |
|---|---|
| 第 1 次：完成 What → 解锁 How | 老学者解释"理解层"是什么、和"认知层"的区别；学徒提问回应 |
| 第 2 次：完成 How → 解锁 Why | 老学者解释因果层的意义：追问动机、因果、反事实 |
| 第 3 次：完成 Why → 解锁 System | 老学者解释系统层的开放性：连接知识、映射现实 |

**对话框区域**：复用 What 阶段的对话框样式，当前说话者头像高亮，对方变暗。

#### 4.2.8 高阶问答引擎（# 8）与诊断反馈（# 9）

核心机制定义为 **AI Cognitive Engine**，定位为"主动维护一条 Socratic Path，系统知道当前节点要训练什么、用户卡在哪里，以及下一问应推向哪个理解层级"。

**核心交互原则**：
> 关卡 NPC 只提出原始谜题；What 由老学者给材料，How / Why / System 由老学者提问；先诊断，再讲解；先让用户思考，再给答案。

老学者在高阶层不应一开始灌输结论。当用户明显不知道、回答严重偏离或连续两轮卡住时，需要给出最小必要讲解。讲解必须服务于下一次思考，而不是替用户完成理解。

**提问策略**：
- How 问机制、步骤、过程
- Why 问原因、反事实、底层条件
- System 问跨节点连接、现实迁移、反例
- 第一问不直接给结论
- 不暴露原文答案
- 不提供选项

**业务逻辑**：每个节点在每个高阶深度最多 3 轮交互。
- 回答覆盖当前深度核心要点：当前深度标记为 `completed`，触发迷雾消散
- 回答模糊：输出提示与下一问，当前深度保持 `learning`
- 用户完全不知道：输出最小必要讲解（≤ 120 字），再给一个更低门槛追问
- 连续 3 轮未完成：保留 `learning` 状态，允许用户稍后返回

**稍后返回规则**：
- 用户离开后再次进入同一节点、同一深度时，开启新的 3 轮尝试周期
- 历史回答和反馈保留在该节点记录中，老学者开场时用一句话提醒
- 新周期的 `round` 从 1 开始，UI 不显示"第 4 轮"

**规则约束**：
- 用户回答长度：≤ 500 字
- 单次诊断超时：5 秒，超时回退为"理解中 + 留追问"
- 失败时按 `learning` 处理，保护用户体验
- 最小必要讲解长度：≤ 120 字，必须包含一个后续追问
- 不提供选择题或选项，保持开放自由回答

#### 4.2.9 迷雾消散与解锁（# 10）

**动画分镜**（MVP 表现级）：
1. 节点中心闪光
2. 光圈以节点为中心向外辐射
3. 当前深度迷雾粒子向外扩散并淡出
4. 1 个下一节点从迷雾中浮现（弹性出现）
5. HUD 的"认知迷雾已消散"数值轻微跳动

**首次提示**：用户第一次看到迷雾散去时，老学者说"迷雾散去了一部分——对前一个节点的理解让你看见了相邻的问题。"该提示只出现一次。

#### 4.2.10 认知进度模型（# 11）

每个节点记录四个深度的进度状态：

| 字段 | 含义 | 示例 |
|---|---|---|
| `introScene` | 小场景观看状态 | `unseen` / `seen` |
| `scrollChoice` | What 卷轴选择（从三张中选择） | `definition` / `example` / `bridge` |
| `what` | 节点 What 深度状态 | `locked` / `available` / `completed` |
| `how` | 节点 How 深度状态 | `locked` / `available` / `learning` / `completed` |
| `why` | 节点 Why 深度状态 | `locked` / `available` / `learning` / `completed` |
| `system` | 节点 System 深度状态 | `locked` / `available` / `learning` / `completed` |
| `finalQuestion` | 原问回响状态 | `locked` / `available` / `completed` |
| `nodeClear` | 节点完整通关状态 | `false` / `true` |

状态流转：`locked → available → learning → completed`

当 `what`、`how`、`why`、`system` 全部为 `completed` 时，`finalQuestion` 从 `locked` 变为 `available`。用户完成原问回响后，节点完整通关。

HUD 文案为"认知迷雾已消散 X%"，不出现考试感表述。

#### 4.2.11 原问回响（# 12）

**触发条件**：同一节点的四个深度均为 `completed`。

**流程**：
1. System 深度完成后，老学者弹出引导文案
2. 系统提示用户可选择"现在回答"或"稍后回来"
3. 用户选择"现在回答"时，地图自动切回 What 层，聚焦该节点的关卡 NPC
4. 关卡 NPC 再次提出原始 `mysteryQuestion`，不得另生成新问题
5. 用户用自己的话回答；系统判断是否覆盖概念准确、机制完整、原因解释、迁移意识四个维度
6. 通过后，关卡 NPC 给出一句简短认可；节点进入完整通关状态

**边界规则**：
- 关卡 NPC 只说原始谜题和最终认可，不输出解释、提示或追问
- 终问文本必须等于该节点的 `mysteryQuestion`
- 用户选择"稍后回来"时，节点在 What 地图显示可答标记

## 5. 风险与降级方案

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| LLM 评估不一致（用户答对被判错） | 高 | 高 | 引入"通过 + 留追问"软判定，避免硬性拒绝；提供"我觉得我答对了"申诉按钮（MVP 后接入） |
| AI 反馈过度，重新变成讲课工具 | 中 | 高 | What 层不诊断；How / Why / System 限制最小讲解长度 ≤ 120 字；每次讲解后必须接追问 |
| 生成耗时过长（>2 分钟） | 中 | 高 | 预制世界兜底；生成期展示等待动画掩盖等待 |
| 用户不理解四张地图关系 | 中 | 高 | 节点坐标保持不变；深度切换器标注锁定/可进入/已完成；首次切换弹出卷轴选择界面引导 |
| 用户把迷雾散去理解成奖励解锁，而不是发现相邻问题 | 中 | 中 | 每次完成节点只显露 1 个下一节点；首次迷雾散去后由老学者解释 |
| 小场景重复播放造成打扰 | 中 | 中 | 仅状态为 `unseen` 时自动播放，播放完成立即标记 `seen` |
| 像素地图布局重叠 | 中 | 中 | 节点数硬上限 20；预制世界手工校准坐标 |
| 评委 30 秒内 get 不到产品价值 | 中 | 极高 | Demo 视频前 10 秒必须出现"迷雾消散"镜头 |

## 7. 验收标准

MVP 必须同时满足以下条件方可视为可参赛：

- 《人类简史》预制世界开箱可玩，至少包含 6 个节点，每个节点具备 What / How / Why / System 四个深度内容
- 地图上节点默认显示谜题标题与关卡 NPC 头像，不显示节点名
- 首次进入节点 What 层时，先播放小场景，再由关卡 NPC 展示原始谜题，随后进入学者-老学者引导对话（约 5 句）
- 对话结束后，三张卷轴在对话框上方逐一水平展开，对话框同步显示老学者讲解
- 卷轴阅读完毕后有简短收尾对话，然后进入轻量确认：用户从三张卷轴中选择 1 张
- 至少 3 个 P0 节点具备 5-8 秒小场景引入；同一节点再次访问不再触发
- What 地图中，用户能完成至少 3 个节点，并看到迷雾每次只显露 1 个下一节点
- 第一次迷雾散去时，老学者必须出现提示
- 完成第 1 个 What 节点后，自动弹出四卷轴深度选择界面，包含引导对话
- 点击卷轴可展开查看深度介绍，确认后可进入新深度
- 后续通过 HUD 按钮快速切换深度，不再经过卷轴界面
- 深度切换器正确表达已完成、可进入、锁定三种状态
- 完成 ≥ 1 个 What 节点后能成功进入 How 地图，背景正确切换，节点坐标不变
- 向下切换地图时进度不丢失
- 对话+卷轴阶段提供"跳过引导"选项
- 单个会话内可完成 How 问答→AI 诊断反馈→迷雾消散完整链路
