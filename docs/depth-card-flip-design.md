# 深度切换卡片翻转动画设计方案

> 本文档记录像素风 + 纸艺（Papercraft）混合风格的深度切换交互设计，为 M4 / 复赛阶段实现预留。

文档创建日期：2026-06-17
所属阶段：M4 打磨 / 复赛优化
状态：设计待实现

---

## 1. 设计目标

### 1.1 要解决的问题

当前版本的深度切换采用背景交叉溶解 + 节点弹性显现的简化方案。该方案功能完整但不具备"认知跃迁"的仪式感。本文档描述的卡片翻转动画旨在解决以下产品问题：

| 问题 | 卡牌方案如何解决 |
|---|---|
| 用户切换深度时感受不到"深入了一层" | 每张卡独立的翻转/旋转动作让"切换"有物理重量 |
| 用户难以理解"同一批节点在不同深度" | 卡牌翻面后坐标不变但外观改变，直观展示"同一个东西，更深一层" |
| 地图感觉像一张静态图片而不是"世界" | 卡牌分层 + 独立动画让地图有了纸艺的立体感和手工感 |

### 1.2 核心隐喻

> 知识世界 = 一幅由手工剪裁卡牌拼贴而成的立体纸艺地图
> 
| 概念 | 视觉表达 |
|---|---|
| 每个知识节点 | 一张独立站立的卡牌（带 NPC 头像 + 谜题标题） |
| 每个深度 | 一套卡牌组（节点卡 + 地形卡），共用同一坐标网格 |
| 深度切换 | 当前套卡牌折叠 / 飞散 / 重组为目标层套卡牌 |
| 地图整体 | 卡牌拼贴的纸艺 diorama（立体透视模型） |

### 1.3 风格参考

- 视觉：像素风（32×32 / 16×16 像素密度） + 纸艺（卡片厚度、投影、折痕）
- 手感：卡牌翻转带弹性 overshoot，不追求物理精确，追求手感愉悦
- 色调：严格遵循 PRD §2.2 调色板（暖黄 / 金绿 / 石灰 / 紫蓝）

## 2. 地图渲染架构

### 2.1 卡牌分层体系

整个地图由四层堆叠组成，只有前三层参与卡片翻转动画：

```
Layer 0: 背景基底
  ├── 纯色渐变 / 大块纹理（不参与卡片切换）
  ├── 切换时用 alpha tween 渐变过渡
  └── 作用是提供"坐标没变"的空间锚定

Layer 1: 地形卡（环境装饰）
  ├── 树、房子、山、石头、草丛等非交互元素
  ├── 每张卡 = PixiJS Container（正面 Sprite + 背面 Sprite）
  ├── 切换时: 原地 scaleX 翻折（方式一）
  └── 作用是维持世界感，不抢视线

Layer 2: 节点卡（核心交互）
  ├── 关卡 NPC 头像 + 谜题标题
  ├── 每张卡 = PixiJS Container（NPC Sprite + 标题文字 + 光晕）
  ├── 切换时: 旋转飞起 + 落下（方式二），stagger 以学者所在位置为圆心扩散
  └── 作用是吸引用户注意力，强调"知识点在深入"

Layer 3: 迷雾层
  ├── 紫蓝色迷雾粒子覆盖
  ├── 不参与卡片切换
  └── 切换期间淡出，切换完成后重新按依赖解锁规则渲染
```

### 2.2 卡牌数据模型

```typescript
// 每张卡牌的通用结构
interface CardSprite {
  id: string;                       // 唯一标识
  type: 'terrain' | 'node' | 'scholar';
  x: number;                        // 地图坐标 x
  y: number;                        // 地图坐标 y
  width: number;                    // 卡牌宽度（像素）
  height: number;                   // 卡牌高度（像素）
  pivot: { x: number; y: number }; // 翻转锚点
  
  // 每个深度对应一套外观
  faces: {
    [depth in Depth]: {
      frontSprite: Texture;         // 正面纹理
      backSprite: Texture;          // 背面纹理（可选，默认棕色羊皮纸）
      tint: number;                 // 主色调
      scale: number;                // 基准缩放（配合弹性动画）
    };
  };
  
  // 运行时状态
  currentDepth: Depth;
  isFlipping: boolean;
  completedDepths: Depth[];         // 已完成的深度（用于光晕显示）
}
```

### 2.3 与当前数据结构的兼容

在当前的 `WorldNode` 和 `WorldLayer` 类型基础上，新增两个字段：

```typescript
interface WorldNode {
  // ... 现有字段（id, name, layer, x, y, gateNpc, mysteryQuestion 等）
  
  // 新增：节点卡牌配置
  cardConfig?: {
    width: number;                  // 默认 128
    height: number;                 // 默认 160
    pivot: 'center' | 'bottom' | 'top';  // 翻转锚点
    flipStyle: 'fold' | 'spin';    // 翻转方式，节点卡默认 spin
  };
}

// 新增：地形卡数据（在 World 层扩展）
interface World {
  // ... 现有字段
  
  // 新增：地形装饰卡集合
  terrainCards?: TerrainCard[];
}

interface TerrainCard {
  id: string;
  type: 'tree' | 'house' | 'mountain' | 'rock' | 'bush';
  x: number;                        // 百分比坐标 0-100
  y: number;
  width: number;                    // 卡牌宽
  height: number;                   // 卡牌高
  flipStyle: 'fold';                // 地形卡统一用 fold
  // 每个深度的外观
  assetMap: Record<Depth, string>;  // 每个深度对应的纹理路径
}
```

## 3. 切换动画方案

### 3.1 混合翻转策略

不同卡牌类型采用不同翻转方式，形成视觉层次：

| 卡牌类型 | 切换方式 | 持续时间 | stagger 间隔 | 排序规则 |
|---|---|---|---|---|
| 地形卡（树/房子/山） | 原地纵向翻折（fold） | 350ms | 30ms | 从地图中心向外扩散 |
| 节点卡（NPC/标题） | 旋转飞起 + 落下（spin） | 500ms | 60ms | 以学者当前位置为圆心扩散 |
| 学者化身 | 原地旋转 360° + 光效闪 | 400ms | 最后执行 | 最后一位 |
| 迷雾层 | 淡出 → 重算 → 淡入 | 450ms | 与 fold 同步 | 全程覆盖 |

### 3.2 动画总时序

```
时序轴（总时长 ≈ 900ms，可交互 ≈ 700ms）

        │◄──── 折叠阶段 ────►│◄── 信号 ──►│◄──── 展开阶段 ──────────►│
        │                    │            │                          │
  T0    T0.15               T0.4        T0.45                      T0.9
  │     │                    │            │                          │
  │     │                    │            │                          │
  初始    地形卡开始向中心翻折   地形卡折叠      地形卡反向展开         所有卡落定
  状态    节点卡开始旋转飞起   到最窄/最高    节点卡旋转落下           光晕顺次点亮
                              全屏闪过        展开 + overshoot        HUD 跳动
                              目标层色条                            交互恢复
                              (100ms)
```

### 3.3 分镜详细描述

#### 阶段一：折叠与飞散（T0 ~ T0.4，400ms）

**地形卡（fold）**：
```
每张地形卡绕 Y 轴从边缘向中线压缩：
  scaleX: 1.0 → 0.0
  alpha: 1.0 → 0.6
  
视觉效果：地图像纸模型一样从四周向中间折叠压缩
stagger：从地图边缘向中心，每张间隔 30ms
总耗时：约 350ms
```

**节点卡（spin）**：
```
每张节点卡先向上飘起 + 旋转：
  y: 原位 → 原位 - 40px
  rotation: 0 → 360°（旋转一圈半）
  scale: 1.0 → 1.15（放大错觉）
  alpha: 1.0 → 0.85

视觉效果：知识点像被风吹起的纸片，旋转着浮在空中
stagger：以学者位置为圆心向外扩散，每张间隔 60ms
总耗时：约 400ms
```

**学者化身**：
```
原地轻微上下浮动（wiggle），提示"要变了"
不参与飞散，保持位置锚定
```

#### 阶段二：切换信号（T0.4 ~ T0.45，50ms）

```
全屏覆盖一个窄矩形色条（目标深度的主色）：
  暖黄 → 金绿 / 金绿 → 石灰 / 石灰 → 紫蓝
  
  从屏幕中央横向展开：
  scaleX: 0 → 1, duration: 30ms
  停留 20ms
  scaleX: 1 → 0, duration: 30ms

等效于摄影中的"快门"——用户眼睛被闪一下，再睁开时世界已经变了
```

#### 阶段三：展开与重组（T0.45 ~ T0.9，450ms）

**地形卡**：
```
反向展开：
  scaleX: 0.0 → 1.0（带 5% overshoot）
  alpha: 0.6 → 1.0
  ease: "back.out(1.7)"

此时卡牌正面已经切换为目标深度的纹理
stagger：从中心向边缘展开（与折叠方向相反）
```

**节点卡**：
```
从上方旋转落下：
  y: 原位 - 40px → 原位（overshoot 超过 8px 再弹回）
  rotation: 720° → 0（再转一圈，落定时归零）
  scale: 1.15 → 1.0（带 overshoot 1.05→1.0）
  ease: "back.out(2.0)"

落定后已完成的光晕顺次点亮（stagger 80ms）
```

**学者化身**：
```
原地旋转 360°（200ms）
落脚时一道白色闪光从上往下扫过（100ms）
```

### 3.4 切换完成后的后续动画

| 元素 | 动画 | 时长 |
|---|---|---|
| 已完成节点光晕 | 顺次点亮：alpha 0→1 + scale 0.8→1.0，stagger 80ms | 400ms |
| HUD 迷雾百分比 | 数值跳动：scale 1.0→1.15→1.0 | 300ms |
| 深度切换器 | 当前层图标脉冲呼吸一次 | 600ms |
| 地图交互 | 恢复 pointer-events，移除遮罩层 | 瞬间 |

### 3.5 回访（向下切换）的动画变体

当用户从深层切回浅层（如 Why → What），动画节奏做以下调整：

| 变化 | 原因 |
|---|---|
| 总时长缩短到 **600ms** | 向下自由，降低等待感 |
| 取消"旋转飞起"阶段，只做原地翻折 | 回访是复习，不需要仪式感 |
| stagger 间隔缩短到 30ms | 加速执行 |
| 已完成节点光晕**先于地形卡展开** | 让用户最先看到"我之前的进度还在" |

## 4. 像素风纸艺视觉细节

### 4.1 卡牌外观规范

| 元素 | 设计 | 像素尺寸 |
|---|---|---|
| 卡牌正面 | 像素纹理 + 1px 深色描边（#1a1226） | 依内容定 |
| 卡牌背面 | 棕色羊皮纸纹理 + 斜向编织纹（4px 间隔） | 与正面一致 |
| 卡牌厚度 | 右侧 + 下侧 2px 深色投影 | 2px |
| 折痕线 | 翻折 50% 时中线出现 1px 白色高光 + 1px 深色阴影 | 1px |
| 卡角 | 2px 像素圆角 + 轻微磨损（角上有随机像素缺失） | 2px |
| 站立阴影 | 卡牌底部椭圆投影（alpha 0.3），跟随卡牌 scale | 椭圆 |

### 4.2 翻转中的关键帧细节

**fold 翻转的中间态**：
```
      ┌──────┐                  ┌──────┐
      │      │                  │      │
      │  正面 │   scaleX=1.0    │  正面 │   scaleX=0.5
      │      │                  │      │
      └──────┘                  └──┐  ┌─┘
                                   │  │    ← 中线已出现折痕
                                   └──┘
           
      ┌─┐                         ┌──────┐
      │ │     scaleX≈0（最薄）    │      │
      │折│    纯色窄条            │ 新正面│  scaleX=展开中
      │痕│                        │      │
      └─┘                         └──────┘
```

**spin 旋转的弧线轨迹**：
```
                    ↗   节点卡飞起路径
                  ↗
                ↗
      ┌─────┐  ↗
      │ NPC │
      │ 初始│
      └─────┘

落回时带弹性：
      ┌─────┐
      │ NPC │  → 弹到上方 8px → 落回原位 → 轻微颤动 2 次完
      └─────┘
```

### 4.3 学者化身的特殊处理

学者化身不参与折叠或飞散，而是在切换过程中：

1. 原地做一个**收伞 → 转伞 → 撑伞**的动画（16×24px 像素动画）
2. 切换到目标层后，伞的颜色变成目标层主题色
3. 如果用户当前在一个已完成节点旁，学者脚下出现一个微型的金色光圈

这个细节很小但记忆点强——用户每次切换深度看到"学者撑伞转了一圈，伞变色了"，很快就能建立"伞色 = 当前深度"的视觉关联。

## 5. 性能与降级

### 5.1 性能预算

| 指标 | 目标 | 硬上限 |
|---|---|---|
| 单次切换总时长 | 800ms | 1000ms |
| 可交互恢复时间 | 600ms | 800ms |
| 卡牌数（地形 + 节点） | ≤ 60 张 | 100 张 |
| 粒子数（切换期间） | ≤ 100 个 | 200 个 |
| 帧率（1080p） | ≥ 30fps | 24fps |

### 5.2 降级方案

| 设备能力不足 | 降级行为 |
|---|---|
| GPU 不支持 WebGL | 跳过卡牌动画，回退到交叉溶解 |
| 卡牌数 > 80 | 地形卡跳过翻转，只做 alpha fade |
| 帧率连续 3 帧 < 24fps | 自动关闭 stagger，所有卡牌同时翻转 |
| 切换期间用户再次点击切换 | 忽略，动画完成后才响应下一次切换 |

### 5.3 实现注意点

```
PixiJS + GSAP 实现要点：
  
  1. 每张卡牌是一个 Container，内含 front/back 两个 Sprite
     切换时只对 Container 做 transform，不操作内部 Sprite
  
  2. 切换前预先加载目标层的所有纹理
     预加载完成才允许触发切换动画
  
  3. 动画期间地图层设置 interactive = false
     GSAP onComplete 回调中恢复
  
  4. stagger 用 GSAP 的 stagger 参数，不手写 setTimeout
     gsap.to(cards, { rotation: 360, duration: 0.5, stagger: { each: 0.06, from: "center" } })
```
