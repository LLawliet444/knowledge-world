# 深度卷轴选择器设计方案

> 在每次新深度首次解锁时，展示一个四卷轴竖排悬挂的选择界面，搭配学者-老学者引导对话，让用户理解"进入新认知维度"的仪式感。

文档创建日期：2026-06-18
状态：设计待实现

---

## 1. 设计目标

### 1.1 要解决的问题

当前深度切换采用 HUD 右下角的 ▲/▼ 箭头按钮直接切换。该方案功能完整但没有"认知跃迁"的仪式感。主要问题：

| 问题 | 说明 |
|------|------|
| 用户首次解锁新深度时无引导 | 系统静默解锁 How 入口，用户不知道"理解层"是什么、和 What 层有什么区别 |
| 四个深度之间的关系不直观 | 用户看不到 Why 和 System 的存在和它们的意义 |
| 缺乏"深入一层"的认知重量 | 从一个深度跳到另一个深度毫无过渡感 |

### 1.2 核心隐喻

> 知识世界的四个深度 = 四幅独立悬挂的认知卷轴

| 概念 | 视觉表达 |
|------|---------|
| 每个认知深度（What/How/Why/System） | 一幅竖挂卷轴，顶部有绳带和轴头 |
| 已解锁深度 | 色彩鲜明、可交互的卷轴 |
| 锁定深度 | 灰色调、锁链缠绕的卷轴 |
| 当前点击的深度 | 卷轴水平展开，展示该深度的介绍文本 |

## 2. 触发规则

### 2.1 触发时机

```
[用户完成某个节点的当前深度]
        ↓
迷雾散去 + 显露邻接节点
        ↓
系统检测：下一深度是否首次解锁？
        ↓
是 → 触发「四卷轴深度入口」界面
否 → 正常完成流程（不展示卷轴界面）

后续自愿切换 → 使用 HUD ▲/▼ 按钮，不再经过卷轴界面
```

### 2.2 具体触发条件

| 触发次数 | 条件 | 展示内容 |
|---------|------|---------|
| 第 1 次 | 用户完成第 1 个节点的 What 层 | What(已通关) + How(可进入) + Why(锁定) + System(锁定) |
| 第 2 次 | 用户完成第 1 个节点的 How 层（且 Why 尚未被介绍过） | What(已通关) + How(已通关) + Why(可进入) + System(锁定) |
| 第 3 次 | 用户完成第 1 个节点的 Why 层（且 System 尚未被介绍过） | What(已通关) + How(已通关) + Why(已通关) + System(可进入) |

**注：** 每个深度只触发一次入口展示。如果用户在 How 层完成了多个节点后才解锁 Why，仍然只触发一次 Why 的介绍。

### 2.3 关闭后行为

```
用户确认进入新深度 →
  关闭卷轴界面
  触发深度切换（复用现有 switchDepth 流程）
  深度切换动画（背景过渡 + 节点状态重算）
  进入新深度地图
  HUD 更新为新深度的状态
```

### 3.2 四卷轴竖挂设计

每个卷轴竖向排列，从顶部到底部依次为 What → How → Why → System，尺寸统一（约 280×100px）。

#### 卷轴三种视觉状态

**已完成（以 What 为例）：**

```
┌──────────────────────┐
│  ⊙   🌱 认知层       │   ⊙ = 棕色木轴端头
│   What               │   暖黄底色 #d4a037
│   ✓ 已通关           │   金色印戳 ✓
│   ────               │   底部标注完成节点数
│   3 / 3 节点         │
└──────────────────────┘
```

**当前可进入（以 How 为例，第一次解锁时）：**

```
┌──────────────────────┐
│  ◇   ⚙️ 理解层       │   ◇ = 金绿色呼吸发光轴端
│   How               │   金绿底色 #7cb342
│   点击展开  →        │   边缘发光动画
│                       │   translateY 浮动 2px
└──────────────────────┘     持续呼吸
```

**锁定（以 Why 为例）：**

```
┌──────────────────────┐
│  ⊠   🔒 因果层       │   ⊠ = 灰色锁定轴端
│   Why               │   灰底色 #9e9e9e
│   未解锁              │   锁链装饰（CSS 伪元素）
│                       │   解锁条件文案悬停显示
│   [悬停：先在 How    │
│    完成 1 个节点]     │
└──────────────────────┘
```

### 3.3 点击当前可进入卷轴 → 展开效果

点击 How/Why/System 卷轴后，该卷轴向左右两侧水平展开，展示深度介绍：

```
┌──────────────────────────────────────────────────┐
│                                                   │
│     ⚙️  How · 理解层                               │
│                                                   │
│  如果说 What 层回答"它是什么"，                    │
│  那么 How 层要回答的是：                          │
│                                                   │
│           "它是如何运作的？"                       │
│                                                   │
│  在这个深度，老学者会给你提出开放问题，            │
│  你需要用你自己的话来回答。                      │
│  不用担心对错——重要的是构建你的理解。            │
│                                                   │
│  下层机制：                                      │
│  · 老学者提问 → 你自由回答 → AI 诊断反馈          │
│  · 每个节点最多 3 轮对话                         │
│  · 回答覆盖核心要点即可通过                       │
│                                                   │
│  模式：问答                                      │
│                                                   │
└──────────────────────────────────────────────────┘
```

#### 展开动画

| 属性 | 值 |
|------|-----|
| 动画 | CSS transform: scaleX(0) → scaleX(1)，从中心向两侧 |
| 时长 | 600ms |
| 缓动 | ease-out |
| 其他 | 配合透明度 0→1，背景淡入 |

#### 展开后底部操作栏

```
┌──────────────────────────────────────────────────┐
│                                                   │
│  ───────────── 以上为展开后的卷轴内容 ───────────  │
│                                                   │
│                                                   │
├───────────────────────────────────────────────────┤
│  🧙 老学者："准备好了吗？"                         │
│                                                   │
│        [ 好的，出发！→ ]  [ 等等，我再看看 ]       │
└───────────────────────────────────────────────────┘
```

| 按钮 | 行为 |
|------|------|
| 好的，出发！→ | 关闭卷轴界面 → 触发 switchDepth → 进入新深度 |
| 等等，我再看看 | 关闭卷轴展开状态，回到四卷轴列表 |

### 3.4 对话与卷轴界面联动

- **场景打开时**：首次展示四卷轴，底部对话区自动播放引导对话
- **用户点击卷轴展开时**：对话暂停，焦点移到展开卷轴的内容
- **用户点击"好的，出发"时**：对话区可选播放一句简短的过渡语，然后执行切换
- **用户停留在卷轴界面时**：对话区显示老学者的引导提问，可点击推进

## 4. 三次引导对话全文

### 4.1 第 1 次：学习如何 → 解锁 How

**触发：** 完成第 1 个节点的 What → How 首次解锁

| # | 说话者 | 台词 |
|---|--------|------|
| 1 | 学者 | "我完成了第一个节点的认知层……好像只打开了表面。" |
| 2 | 老学者 | "你直觉很准。What 层只是第一步——让你知道'它是什么'。现在你解锁了第二层：How。" |
| 3 | 学者 | "How 层……是要我回答'它如何运作'？" |
| 4 | 老学者 | "没错。这里不再有现成的卡给你读。我会给你提出问题，你需要用自己的话来回答。不用怕答错——重要的是开始思考。" |
| 5 | 学者 | "感觉比 What 层要难……但也更有意思了。" |
| 6 | 老学者 | "准备好了就打开那卷 How 卷轴，我们进去看看。" |

**How 卷轴展开后的老学者收尾：**

> 🧙 老学者："准备好了吗？"

---

### 4.2 第 2 次：解锁 Why

**触发：** 完成第 1 个节点的 How → Why 首次解锁

| # | 说话者 | 台词 |
|---|--------|------|
| 1 | 学者 | "我答完了 How 层的问题……现在我能看到更深一层了？" |
| 2 | 老学者 | "是的。Why 层是探索'为什么'的维度。如果说 How 是理解机器的齿轮，Why 就是追问这台机器为什么被造出来。" |
| 3 | 学者 | "它追问的是因果关系和根本原因？" |
| 4 | 老学者 | "正是。这一层需要你跳出刚才的机制思维，去追问动机、因果、反事实——'如果不是这样，会怎样？'打开了就明白了。" |

**Why 卷轴内容：**

```
🔍  Why · 因果层

What 告诉你"它是什么"
How 告诉你"它如何运作"
而 Why 要追问的是：

         "它为什么是这样？"

在这个深度，你需要跳出机制本身，
去追问背后的因果关系、动机、以及反事实：
"如果历史没有这样发展，世界会有什么不同？"

下层机制：
· 老学者提出为什么问题 + 反事实假设
· 你需要用因果链条解释
· 覆盖核心原因即可通过

模式：深层问答
```

---

### 4.3 第 3 次：解锁 System

**触发：** 完成第 1 个节点的 Why → System 首次解锁

| # | 说话者 | 台词 |
|---|--------|------|
| 1 | 学者 | "Why 层让我看到了背后的原因……还有更深的一层？" |
| 2 | 老学者 | "System 层是最后一层——也是最开阔的一层。它让你把刚学到的知识放到更大的系统中去看：它如何与其他知识连接、如何映射到你的现实生活。" |
| 3 | 学者 | "所以 System 层是连接一切的地方……" |
| 4 | 老学者 | "是的。到这里不再有标准答案，只有你自己的理解网络。打开 System 卷轴吧。" |

**System 卷轴内容：**

```
🌌  System · 系统层

What: 它是什么
How:  它如何运作
Why:  它为什么是这样
而 System 要追问的是：

        "它还连接了什么？"

在这个最开阔的深度，你被邀请站在更高的视角：
这个知识和你已知的其他知识如何连接？
它如何映射到你现实生活中的问题？

下层机制：
· 老学者提出跨领域连接问题
· 鼓励迁移到现实场景
· 不需要"标准答案"，需要你自己的理解和连接

模式：自由探索 + 迁移
```

## 5. 视觉与动画规范

### 5.1 卷轴样式

| 元素 | 设计 |
|------|------|
| 轴端（顶部） | 像素风木制圆轴端头（6×6），颜色随状态变化 |
| 轴端（底部） | 同样式，稍小（4×4） |
| 卷轴本体 | 浅底 + 纹理，圆角 4px |
| 顶部绳带 | 深色像素绳带，锁定状缠绕时为 2px 斜线纹 |
| 右侧解锁标签 | "点击展开 →" 带右箭头动画（左右 wiggle） |
| 锁定状态 | 灰色 50% 透明度，斜向条纹覆盖 + 🔒 图标 |

### 5.2 色彩系统

| 深度 | 已完成色 | 可进入色 | 锁定色 |
|------|---------|---------|-------|
| What | #d4a037 暖金 | — | — |
| How | #7cb342 金绿 | #8bc34a 亮绿（呼吸发光） | — |
| Why | #6b5b95 紫灰 | #7e57c2 紫蓝（呼吸发光） | #9e9e9e |
| System | — | — | #9e9e9e |

### 5.3 动画规范

| 动画 | 触发 | 时长 | 说明 |
|------|------|------|------|
| 卷轴呼吸发光 | 可进入卷轴常驻 | 持续 | 边缘 glow pulse + translateY 上下浮动 2px，2000ms 周期 |
| 卷轴展开 | 点击可进入卷轴 | 600ms | scaleX(0→1) 从中心向两侧，ease-out |
| 卷轴收卷 | 点击"等等"或切换 | 400ms | scaleX(1→0) 从两侧向中心 |
| 页面淡入 | 界面打开 | 500ms | opacity 0→1 + scale 0.95→1 |
| 页面淡出 | 确认切换 | 400ms | opacity 1→0 |
| 锁链解除动画 | 首次触发时锁定卷轴 | 1000ms | 锁链碎裂消散（CSS 粒子效果，仅第 1 次展示） |

## 6. 实现技术变更

### 6.1 新建文件

| 文件 | 内容 |
|------|------|
| `apps/frontend/src/components/depth/DepthScrollSelector.tsx` | 深度卷轴选择器主组件：四卷轴竖排渲染 + 展开状态 + 对话区 + 操作按钮 |
| `apps/frontend/src/components/depth/DepthScrollItem.tsx` | 单张深度卷轴组件：三种状态渲染 + 呼吸/发光动画 + 展开/收卷动画 |

### 6.2 修改文件

| 文件 | 变更 |
|------|------|
| `apps/frontend/src/types/world.ts` | 无新增类型（复用现有 LayerType） |
| `apps/frontend/src/store/worldStore.ts` | 新增 `shownDepthGuides: Set<LayerType>` 状态，跟踪已展示的引导；`switchDepth` 逻辑扩展：检测是否需要先展示卷轴界面 |
| `apps/frontend/src/store/dialogStore.ts` | 新增 `"depth_scroll"` DialogPhase；新增 `startDepthScroll()` action；存储即将切换的目标深度 |
| `apps/frontend/src/components/map/WorldMap.tsx` | 在 `handleSceneComplete` 或 `handleWhatComplete` 逻辑中新增：完成节点后检测新深度解锁 → 调用 `startDepthScroll()` |
| `apps/frontend/src/App.tsx` | 或 `DepthScrollSelector` 挂载点，根据 phase="depth_scroll" 渲染（在 DialogBox 同级） |

### 6.3 数据流

```
WorldMap
  │ handleNodeClick → 学者走到节点 → 过场动画 → DialogBox(对话+卷轴)
  │                                                                
  │ 用户完成 What 翻卡（handleWhatComplete）                       
  │ 调用 updateNodeDepthState(nodeId, "what", "completed")        
  │                                                                
  ▼                                                                
WorldMap
  │ 检测：完成第 1 个 What → How 首次解锁？
  │ 是 → set({ shownDepthGuides: prev.add("how") })
  │ 调用 dialogStore.startDepthScroll()
  │ (如果已展示过，正常 close)
  │
  ▼
DepthScrollSelector 
  │ 展示四卷轴界面 + 引导对话
  │ 用户点击"好的，出发！"
  ▼
  → switchDepth(depth) → 深度切换动画 → 进入新深度
```

### 6.4 兼容性

- 如果 `shownDepthGuides` 已经包含所有深度，不展示卷轴界面，走正常 HUD 切换
- 引入 `DepthScrollSelector` 后，**不影响**已有 HUD 的 ▲/▼ 按钮功能
- 回访场景：用户第二次进入世界，已经展示过的引导不再触发

## 8. 验收标准

- [ ] 完成第 1 个 What 节点后，自动弹出深度卷轴选择界面（不直接进入 How）
- [ ] 四卷轴竖排展示：What(已完成) + How(可进入·发光) + Why(锁定) + System(锁定)
- [ ] 底部对话区显示第 1 次引导对话（学者↔老学者，6 句）
- [ ] 点击 How 卷轴 → 水平展开显示 How 深度介绍内容
- [ ] 展开后底部出现"好的，出发！"/"等等，我再看看"两个按钮
- [ ] 点击"好的，出发"→ 关闭卷轴界面 → 深度切换 → 进入 How 地图
- [ ] 点击"等等"→ 卷轴收卷，回到四卷轴列表
- [ ] 完成第 1 个 How 节点后，触发 Why 的首次引导（第 2 次对话 + Why 卷轴展开）
- [ ] 完成第 1 个 Why 节点后，触发 System 的首次引导（第 3 次对话 + System 卷轴展开）
- [ ] 每个深度仅触发一次引导，之后通过 HUD 按钮正常切换
- [ ] 锁定卷轴不可点击，悬停显示解锁条件
- [ ] 回访场景不会再弹出引导
