# Knowledge World — 玩法重构方案 v2

> 基于 2026-06-17 产品讨论，整合四地图深度模型 + GPT 建议

---

## 一、核心模型变更

### 原方案

```
一张地图，四个认知层横带（What | How | Why | System）
节点按认知层分配到不同横带
所有节点在同一画面中展示
```

### 新方案

```
四张独立地图，分别对应 What / How / Why / System 四个认知深度
同一个节点集合出现在所有四张地图上（坐标不变）
用户在同一批节点上反复深入，每深入一层，地图切换一次
```

**地图 ≠ 知识分区，地图 = 认知深度可视化。**

---

## 二、三条规则

| 规则 | 内容 |
|------|------|
| **向下自由** | 用户随时可以切换到任意低层地图（如从 Why 回到 What），无惩罚、无门槛 |
| **向上门禁** | 进入 How 地图需要 ≥1 节点完成 What；进入 Why 需要 ≥1 节点完成 How；进入 System 需要 ≥1 节点完成 Why |
| **依赖解锁** | 在深度 D 下，节点 N 的 D 层内容可用，当且仅当存在一个邻居 M 且 M 在 D 层已完成。startNode 在当前深度进入时自动可用 |

---

## 三、四层交互模式

每一层不只是地图不同，交互方式也不同：

| 层 | 交互类型 | 用户行为 | 每节点耗时 | 难度 |
|----|---------|---------|-----------|------|
| **What** | 阅读翻卡 | 读 3 张卡片 → 点"记住了" → 完成 | ~20-30 秒 | 极低 |
| **How** | 问答 + AI 反馈 | 看问题 → 打字回答 → 收到四象限反馈卡 | ~2-3 分钟 | 中等 |
| **Why** | 追问式问答 | 回答 → 反馈 → AI 继续追问 → 可多轮 | ~3-5 分钟 | 较高 |
| **System** | 跨域问答 | 需要连接不同节点的知识，讨论反例 | ~5 分钟 | 高 |

### What 层 — 翻卡阅读（新增交互）

每个节点的 What 内容拆为 3 张卡片：

```
Card 1 (📜 定义)   → "这是什么东西"
Card 2 (💡 例子)   → "这长什么样"
Card 3 (🌉 连接)   → "这为什么重要（自然引出下一个节点）"
```

用户操作：点击 [下一张 →] 翻卡，最后一张点 [我记住了，继续探索 ✓]。不问答、不打字、不判断对错。

### How / Why / System 层 — 问答（复用现有 DialogBox）

问题风格随深度变化：

| 层 | 提问风格 |
|----|---------|
| How | "描述…是怎么一步步发生的" / "如果没有…会怎样" |
| Why | "为什么是…而不是…" / "…和…之间有什么因果关系" |
| System | "…和…之间有什么共同的底层逻辑" / "找一个反例" |

每轮反馈卡保持四象限：已理解 / 还缺一点 / 提示 / 下一问。用户可选择"继续追问"多轮，直到节点在该深度标记完成。

## 四、地图切换体验

### 视觉

- 背景图从 `world.mapImage` 切换为 `BIOME_IMAGES[currentDepth]`（4 张 biome 图全屏呈现，不再切成横带）
- 节点坐标不变，学者位置不变
- 切换动画：淡出 → 换背景/节点 → 淡入（~600ms）

### 操作

地图角落显示深度切换器：

```
[🌿 What ✓] [🌾 How] [🏔 Why] [🔬 System]
 已完成      可进入      锁定      锁定
```

- 向下的层：直接点击，即时切换
- 向上的层：满足门禁条件后解锁，否则显示锁图标

## 五、新增：谜题标题 + NPC 学者

### 5.1 节点在地图上不显示名字，显示谜题

```
原方案:  地图上显示 "农业革命"
新方案:  地图上显示 "❓ 为什么农民比猎人更累？"
         （NPC 老农夫的头像站在旁边）
```

目的：用户点节点不是因为"该学这个"，而是因为"想知道答案"。

### 5.2 节点图标改为 NPC 学者头像

```
原方案:  节点用齿轮、麦穗、卷轴等抽象图标
新方案:  每个节点对应一个 NPC 学者角色
         "农业革命" → 老农夫 👨‍🌾
         "货币"     → 商人 💰
         "国家"     → 国王 👑
         "宗教"     → 祭司 🙏
```

NPC 在不同深度地图上问不同难度的问题（同一角色，提问随深度变深），世界更有温度。

### 5.3 所需新增数据字段

每个节点新增两个字段：

```typescript
{
  // ...原有字段不变
  mysteryQuestion: "为什么农民比猎人更累？",  // 地图上显示的谜题
  npc: {
    character: "farmer",      // NPC 角色 ID，对应头像图
    title: "老农夫",
    avatar: "/npcs/farmer.png",
  },
  whatCards: [                // What 层翻卡的 3 张卡片
    { type: "definition", text: "..." },
    { type: "example", text: "..." },
    { type: "bridge", text: "..." },
  ],
}
```

## 六、新增：迷雾结构性隐藏节点

### 原方案

迷雾是装饰性粒子层（漂移效果），所有节点始终可见。

### 新方案

- 未在当前深度解锁的节点：**完全不可见**（隐藏在迷雾下）
- 当前层进入时：只显示 startNode
- 完成一个节点：迷雾消散动画 → 露出邻居节点及其谜题
- 节点可见范围始终是"当前层你已解锁的局部"，而非全局

效果：玩家永远有"迷雾后面是什么"的好奇心，推动继续探索。

## 七、HUD 文案变更

```
原方案:  "已掌握 3 / 7 节点"
新方案:  "认知迷雾已消散 43%"
```

与产品叙事一致：不是"学了多少"，而是"世界清晰了多少"。

## 八、数据模型的变更范围

### 不改动部分

- `World`、`WorldNode`、`WorldLayer`、`LayerType`、`NodeState` 的类型定义**不变**
- `World.layers` 结构不变（仍按 what/how/why/system 分组）
- `BIOME_IMAGES`、`NODE_STYLES` 等常量**不变**

### 新增部分

| 新增 | 说明 |
|------|------|
| `progress: Record<string, CognitiveProgress>` | 每个节点四深度的完成状态（what/how/why/system 各为 "locked" / "completed"） |
| `currentDepth: LayerType` | 当前所在的认知深度 |
| `node.mysteryQuestion` | 地图上显示的谜题文本 |
| `node.npc` | NPC 角色信息（角色 ID、称号、头像图路径） |
| `node.whatCards` | What 层翻卡内容（3 张卡片） |

### 修改部分

| 位置 | 改动 |
|------|------|
| `App.tsx` | 新增 `currentDepth` 和 `progress` 状态管理；地图切换 UI |
| `WorldMap.tsx / MapRenderer.ts` | 按 `currentDepth` 层筛选节点并过滤可用的；背景图改用 `BIOME_IMAGES[currentDepth]` |
| `DialogBox.tsx` | 新增 `mode="reading"` 翻卡模式 |
| `client.ts`（mock） | 问答 prompt 按 `layer` 调整风格 |

## 十、一个节点的完整生命周期（全流程示例）

以《人类简史》的"认知革命"为例：

```
Step 1  │ 进入 What 地图（草原背景）
        │ 地图上看到：❓ "是什么让智人和其他动物不同？"
        │           👴 长老 站在旁边
        │ 点击 → 学者走过去 → 弹出翻卡对话框
        
Step 2  │ 📜 "约七万年前，智人开始拥有虚构故事的能力..."
        │ 💡 "比如'公司'、'法律'、'货币'..."
        │ 🌉 "这让智人突破了 150 人社交上限..."
        │ [我记住了，继续探索 ✓]
        
Step 3  │ 节点标记 What ✅
        │ 迷雾消散 → 露出邻居"用火控制"的谜题和 NPC
        │ 点击下一个节点，或手动切换到 How 地图
        
Step 4  │ 切换到 How 地图（背景从草原变成森林）
        │ "认知革命"仍在地图同一位置，NPC 还是同一个长老
        │ 但这次他问的是："认知革命是如何改变社会组织结构的？"
        │ 用户打字回答 → AI 反馈卡 → 继续追问或完成
        
Step 5  │ 节点标记 How ✅
        │ 继续走 Why / System 地图
        │ 同一 NPC，问题越来越深
        
Step 6  │ 四个深度全部完成 → 节点显示紫色光晕
        │ "认知革命"在全世界的理解达到完整
```

## 十一、与初赛 MVP 范围的对照

| MVP 要求 | 本方案如何覆盖 |
|----------|---------------|
| 至少 1 个预制世界全可玩 | 一个世界 6-7 节点 × 4 深度 = 24-28 次交互，体量足够 |
| 迷雾消散动画（前 10 秒视觉钩子） | 迷雾结构性隐藏 + 每次解锁触发消散，比原方案出现更频繁 |
| 节点点击 → 学者移动 → NPC 对话框 | NPC 学者替换抽象图标，有角色有温度 |
| AI 反馈卡四段式 | How/Why/System 层复用现有反馈卡，What 翻卡不打乱节奏 |
| 评委 30 秒内理解"知识迷雾"机制 | "世界清晰度" + 迷雾消散 + 谜题驱动，30 秒足够留下印象 |
| PDF 上传（P1，演示能力） | 不受影响，后端数据生成即可适配本模型 |
| 第二个预制世界（P1） | 同样适用，数据结构不变 |
